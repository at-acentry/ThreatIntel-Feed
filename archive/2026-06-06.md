> **Daily Brief.** The week ending June 6 has been dense with high-severity activity across three distinct threat vectors: exploited-in-the-wild zero-days demanding immediate patch action, concurrent nation-state espionage campaigns from Russia, Iran, and China, and a fresh supply chain compromise affecting Red Hat's npm ecosystem. ## Executive Summary

- **CVE-2026-20245 (Cisco SD-WAN, zero-day, no patch):** A seventh Catalyst SD-WAN zero-day disclosed June 5, enabling root command execution via crafted CLI file uploads by authenticated netadmin-level users. Actively exploited; no remediation available at time of disclosure. Mandiant credited with discovery.
- **CVE-2025-48595 (Android Framework, KEV, zero-day):** Google confirmed limited targeted exploitation of an integer overflow in Android 14+ allowing local privilege escalation. Added to CISA KEV June 2 with a June 5 federal deadline. June 2026 Android security patch resolves this.
- **APT28 (Fancy Bear) CVE-2026-21513:** Russian GRU-linked APT exploited an MSHTML zero-day prior to the February 2026 Patch Tuesday, targeting Ukraine and EU governments. An incomplete patch led to follow-on CVE-2026-32202, keeping residual risk elevated.
- **Screening Serpens (Iran, UNC1549):** Unit 42 documented six new RAT variants (MiniUpdate and MiniJunk V2 families) deployed Feb–Apr 2026, targeting US, Israeli, UAE, and broader Middle Eastern defense and tech sectors using sophisticated AppDomainManager hijacking to evade EDR telemetry.
- **Red Hat npm supply chain:** A June 1–2 attack via compromised GitHub Actions OIDC pushed credential-stealing Miasma malware (Mini Shai-Hulud variant) across 32 packages and 96 package versions under `@redhat-cloud-services`. 116,000+ accumulated downloads. CI secrets, cloud credentials, SSH keys, and npm tokens in affected pipeline environments should be treated as compromised.
- **Citrix NetScaler CVE-2026-3055 (CVSS 9.3):** Large-scale exploitation of a SAML IDP memory overread confirmed by Fortinet. Unpatched SAML-configured NetScaler appliances should be treated as a priority emergency patch target.

**CVE-2026-20245 — Cisco Catalyst SD-WAN Manager (no patch):** An authenticated attacker with netadmin privileges can upload a crafted file via the CLI to execute arbitrary commands as root. This is the seventh SD-WAN zero-day exploited in 2026; all deployments — on-prem, Cloud-Pro, Cisco Managed, and FedRAMP — are affected. Mandiant identified active exploitation in limited incidents involving unauthorized configuration pushes to edge devices. No patch is available; mitigate through privilege review, audit-log monitoring, and network segmentation.

**CVE-2026-41089 — Windows Netlogon (CVSS 9.8):** A buffer overflow in the Windows Netlogon Remote Protocol that the Belgian government has warned is being actively exploited to compromise domain controllers. Patch domain controllers first; exploitation does not require authentication.

**CVE-2025-48595 — Android Framework Zero-Day (CVSS 8.4, KEV):** Integer overflow in Android 14+ enables local privilege escalation. Google confirmed "limited, targeted exploitation" consistent with spyware vendor or nation-state activity. Federal deadline was June 5. Enterprise MDM fleets should push the June 2026 Android patch immediately; apply conditional access restrictions for unpatched devices while rollout completes.

**CVE-2026-3055 — Citrix NetScaler (CVSS 9.3):** Out-of-bounds memory read triggered by crafted SAML requests to `/saml/login`. Attackers leak memory contents — potentially session tokens and credentials — via the `NSC_TASS` cookie. Fortinet confirmed large-scale exploitation; exploitation requires the appliance to be configured as a SAML IDP. Patch immediately per Citrix bulletin CTX696300.

- **Targets:** Aerospace, defense manufacturing, telecommunications, and technology sector professionals in the US, Israel, UAE, and broader Middle East; individual targeting via spear-phishing aligned with job-seeker profiles

The MiniUpdate family introduced advanced AppDomainManager hijacking that preemptively disables ETW and bypasses strong-name validation — an evolution not previously observed from this group.
- **Attribution confidence:** Moderate-High — Unit 42 attribution based on shared tooling, sideloading chain signatures, ONLYOFFICE infrastructure overlap, and prior reporting by Check Point and ClearSky on the same group
- **Sources:** [Unit 42](https://unit42.paloaltonetworks.com/tracking-iran-apt-screening-serpens/), [The Hacker News](https://thehackernews.com/2026/05/cisa-adds-exploited-langflow-and-trend.html)

New targets include universities in Argentina and Mexico via compromised Cisco routers, a Myanmar-based telecom (Mytel), and a South African telecom provider.
- **Attribution confidence:** High — attributed by CISA, FBI, Mandiant, and Trend Micro; subject of US Treasury sanctions and Congressional Research Service reporting
- **Sources:** [TechCrunch](https://techcrunch.com/2026/03/09/salt-typhoon-china-who-has-been-hacked-global-telecom-giants/), [CISA Advisory AA25-239A](https://www.cisa.gov/news-events/cybersecurity-advisories/aa25-239a), [CyberScoop](https://cyberscoop.com/fbi-salt-typhoon-ongoing-threat-cybertalks-2026/)

## Software Supply Chain Attacks

> Incidents, campaigns, and vulnerabilities affecting software build pipelines, open-source packages, CI/CD infrastructure, third-party libraries, and code-signing infrastructure — reported within the last 7 days.

### @redhat-cloud-services npm Packages — Miasma Credential-Stealing Worm (June 1–2, 2026)

- **Attack type:** Compromised maintainer account / CI/CD pipeline injection via GitHub Actions OIDC
- **Ecosystem:** npm
- **Package(s) affected:** 32 packages across the `@redhat-cloud-services` npm namespace; 96 versions identified as compromised; accumulated downloads exceeding 116,000
- **Impact:** A 4.2 MB obfuscated payload named Miasma (a variant of Mini Shai-Hulud malware, open-sourced by Team PCP) executes at install time via an npm `preinstall` hook. The payload harvests CI secrets, cloud provider credentials (AWS, GCP, Azure), SSH keys, npm tokens, and GitHub credentials from the build environment. Researchers at Aikido Security and Wiz identified the compromise; Wiz attributed the payload to the Mini Shai-Hulud family. Red Hat confirmed the packages are limited to internal development tooling and are not distributed through `console.redhat.com`. Red Hat confirmed no impact to customer or production environments was identified, though investigation is ongoing.
- **IOCs:** Compromised packages under `@redhat-cloud-services` scope; payload artifacts reference repository names containing "Miasma: The Spreading Blight." The compromise was introduced via a GitHub account push to the `RedHatInsights` GitHub organization, authenticated through GitHub Actions OIDC (not a stolen npm token), indicating the attacker had GitHub account or CI pipeline-level access.
- **Mitigation:** Any CI/CD pipeline that installed an `@redhat-cloud-services` package since June 1, 2026, should immediately: (1) rotate all CI secrets, cloud credentials, SSH keys, and npm tokens; (2) audit GitHub Actions OIDC token permissions for least privilege; (3) review npm audit locks and regenerate lock files; (4) enable npm provenance verification and require publish attestation in CI workflows going forward.
- **Source:** [Cybersecurity Dive](https://www.cybersecuritydive.com/news/dozens-red-hat-npm-packages-supply-chain-attack/821723/), [Wiz Blog](https://www.wiz.io/blog/miasma-supply-chain-attack-targeting-redhat-npm-packages), [Aikido Security](https://www.aikido.dev/blog/red-hat-npm-packages-compromised-credential-stealing-worm)
