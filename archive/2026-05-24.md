### @tanstack/* npm Packages — Mini Shai-Hulud Worm (TeamPCP)

- **Attack type:** CI/CD pipeline injection via GitHub Actions cache poisoning; pull_request_target workflow abuse enabling cross-fork cache write
- **Ecosystem:** npm (primary); PyPI (secondary spread)
- **Package(s) affected:** 42 `@tanstack/*` packages (84 malicious artifacts published 2026-05-11 19:20–19:26 UTC); spread to Mistral AI, UiPath, OpenSearch (~1.3M weekly npm downloads), Guardrails AI, and 130+ additional maintainers; 170+ npm packages, 2 PyPI packages total; 404 malicious versions documented; 15M+ weekly downloads at risk
- **Impact:** Malware harvests GitHub tokens, npm publish tokens, CI/CD pipeline secrets, cloud credentials (AWS, GCP, Azure), and API keys from build environments. Achieves persistence on macOS via `~/Library/LaunchAgents/com.user.gh-token-monitor.plist` and on Linux via `~/.config/systemd/user/gh-token-monitor.service`. Exfiltrates harvested secrets over Session/Oxen messenger file-upload network. First documented case of a malicious npm package carrying valid SLSA provenance.
- **IOCs:** C2: `filev2.getsession.org`, `seed1.getsession.org`, `seed2.getsession.org`, `seed3.getsession.org`, `api.masscan.cloud`; typosquatting domain: `git-tanstack.com`; malicious IP: `83.142.209.194`; persistence artifacts: `router_runtime.js`, `setup.mjs` in `.claude/` or `.vscode/` directories; CVE-2026-45321 / GHSA-g7cv-rxg3-hmpx
- **Mitigation:** Audit `package-lock.json` and `yarn.lock` immediately for any `@tanstack/*` version published 2026-05-11; pin to verified clean versions per TanStack incident postmortem; rotate all CI/CD secrets, npm tokens, and cloud credentials if pipelines consumed affected versions; check for persistence LaunchAgent/systemd unit; block `filev2.getsession.org`, `api.masscan.cloud`, and `83.142.209.194` at egress; enable `ACTIONS_PREVENT_UNSECURE_COMMANDS=true` and pin GitHub Actions to commit SHAs; run `npm audit` and `pip-audit` across all projects
- **Source:** [Wiz Blog](https://www.wiz.io/blog/mini-shai-hulud-strikes-again-tanstack-more-npm-packages-compromised), [Tenable FAQ](https://www.tenable.com/blog/mini-shai-hulud-frequently-asked-questions), [TanStack Postmortem](https://tanstack.com/blog/npm-supply-chain-compromise-postmortem), [StepSecurity](https://www.stepsecurity.io/blog/mini-shai-hulud-is-back-a-self-spreading-supply-chain-attack-hits-the-npm-ecosystem), [NHS Digital CC-4781](https://digital.nhs.uk/cyber-alerts/2026/cc-4781)
