> **Daily Brief.** Today's threat landscape is dominated by two converging pressures: a fresh wave of CISA KEV additions targeting enterprise edge devices and e-commerce platforms, and a continuing escalation in state-sponsored pre-positioning against U.S. critical infrastructure. The Iranian ICS/PLC campaign documented in CISA AA26-097A continues to generate operational disruptions across energy and water sectors. Simultaneously, the Miasma supply chain campaign—a successor to the Shai-Hulud family—targeting Red Hat npm packages persists as a high-priority threat for CI/CD environments. Security teams should prioritize patching CVE-2026-35616 (FortiClient EMS) and CVE-2026-45247 (Magento) while auditing npm dependency trees for @redhat-cloud-services packages.

## Executive Summary

- CISA added CVE-2026-45247 (Mirasvit Full Page Cache Warmer for Magento 2, CVSS 9.8) to KEV on June 3, requiring federal patch by June 6; unauthenticated RCE via PHP object deserialization is actively exploited against gaming and retail e-commerce targets in the US, UK, France, and Australia.
- CVE-2026-35616 (Fortinet FortiClient EMS, CVSS 9.8) continues to be exploited to deliver the EKZ infostealer disguised as a Fortinet patch; over 2,000 exposed EMS instances remain unpatched per Shadowserver scans.
- Critical WordPress plugin CVE-2026-3300 (Everest Forms Pro, CVSS 9.8) has seen 29,300+ blocked exploit attempts since April; threat actors are creating rogue administrator accounts on compromised sites.
- The Miasma campaign (Mini Shai-Hulud variant) compromised 31 Red Hat npm packages in the @redhat-cloud-services scope on June 1 within 72 seconds, deploying multi-cloud credential harvesting malware targeting CI/CD pipelines.
- Iranian-affiliated APTs under CISA AA26-097A continue targeting internet-exposed Rockwell Automation PLCs across U.S. water, energy, and government sectors; operational disruptions confirmed.
- Volt Typhoon's pre-conflict positioning in U.S. critical infrastructure (water, energy, communications) has intensified per a February 2026 CISA supplementary advisory, with CISA characterizing activity as preparation for potential disruption rather than espionage.

### Miasma — @redhat-cloud-services npm Packages (Mini Shai-Hulud Variant)
- **Attack type:** Compromised maintainer account / CI pipeline injection
- **Ecosystem:** npm
- **Package(s) affected:** 31 packages under the `@redhat-cloud-services` npm scope; malicious versions published June 1, 2026 within a 72-second window. Affected packages include `@redhat-cloud-services/frontend-components`, `@redhat-cloud-services/insights-client`, and 29 additional packages in the same namespace.
- **Impact:** Obfuscated 4.2 MB payload fires on `npm install` via `preinstall` hook. Multi-stage credential harvester sweeps GitHub Actions secrets, AWS/GCP/Azure/Kubernetes tokens, HashiCorp Vault credentials, npm publishing tokens, and CircleCI secrets. Generates uniquely encrypted payload per infection (hash-based detection limited).
- **IOCs:** Unique per-installation payloads; focus on behavioral detection. Hunt for `preinstall` hooks in affected package versions, unexpected outbound HTTPS from CI runners, and anomalous token usage following recent `npm install` of any `@redhat-cloud-services/*` package.
- **Mitigation:** Immediately audit `node_modules` and `package-lock.json` for any `@redhat-cloud-services/*` packages installed on or after June 1, 2026. Roll all CI/CD secrets, cloud credentials, and npm tokens for systems that ran `npm install` on the affected packages. Pin to versions preceding June 1 or await Red Hat's re-signed clean releases. Reference: [RHSB-2026-006](https://access.redhat.com/security/vulnerabilities/RHSB-2026-006).
- **Source:** [Wiz Blog](https://www.wiz.io/blog/miasma-supply-chain-attack-targeting-redhat-npm-packages), [ReversingLabs](https://www.reversinglabs.com/blog/red-hat-cloud-service-npm-packages-backdoored-in-72-seconds), [Microsoft Security Blog](https://www.microsoft.com/en-us/security/blog/2026/06/02/preinstall-persistence-inside-red-hat-npm-miasma-credential-stealing-campaign/), [Snyk](https://snyk.io/blog/miasma-supply-chain-attack-malicious-code-redhat-cloud-services-npm-packages/)
