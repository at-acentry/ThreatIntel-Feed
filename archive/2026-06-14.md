- **Targets:** U.S. critical infrastructure — electric utilities, telecommunications, water systems, transportation hubs; expanding to communications sectors
- **TTPs:** Living-off-the-land binaries (LOLBins), `T1078` (Valid Accounts), `T1021.002` (Remote Services: SMB/Windows Admin Shares), `T1090` (Proxy: Multi-hop proxy via compromised SOHO routers), `T1053` (Scheduled Task/Job), `T1105` (Ingress Tool Transfer)
- **Recent activity:** CISA's February 2026 supplementary advisory documented intensified activity since mid-2025, with new IOCs identified in water and communications sectors. The advisory characterised current activity as consistent with "pre-conflict positioning." The 2026 ODNI Annual Threat Assessment explicitly named Volt Typhoon as a persistent threat to government and private sector networks.
- **Attribution confidence:** High — attributed by CISA, FBI, NSA, Five Eyes partners, and Unit 42
- **Sources:** [CISA China Threat Overview](https://www.cisa.gov/topics/cyber-threats-and-advisories/nation-state-cyber-actors/china), [Unit 42 Threat Brief](https://unit42.paloaltonetworks.com/volt-typhoon-threat-brief/), [CybelAngel 2026](https://cybelangel.com/blog/volt-typhoon/)

### Salt Typhoon (aliases: GhostEmperor, FamousSparrow, UNC2286)

- **TTPs:** `T1190` (Exploit Public-Facing Application), `T1098` (Account Manipulation), `T1119` (Automated Collection), `T1048` (Exfiltration Over Alternative Protocol — custom SFTP client in Golang), SSH tunnelling on non-standard ports, GRE/IPsec covert channels

### Lazarus Group (aliases: APT38, HIDDEN COBRA, Guardians of Peace, ZINC, NICKEL ACADEMY)

- **Targets:** Cryptocurrency exchanges, DeFi platforms, financial institutions, blockchain companies; current focus on Southeast Asian and Taiwanese exchanges
- **TTPs:** `T1566.002` (Spear-phishing via Telegram scheduling lures), `T1055` (Process Injection — RemotePE executes entirely in RAM), `T1140` (Deobfuscate/Decode), `T1562.001` (Impair Defenses: Disable or Modify Tools — ETW patching, Hell's Gate), `T1041` (Exfiltration Over C2 Channel)

Initial access is via Telegram social engineering impersonating employees through fake Calendly/Picktime scheduling links, directing victims to C2 at `aes-secure.net`.
- **Attribution confidence:** High — Bitdefender, SC Media, The Hacker News, corroborated by DPRK nexus indicators
- **Sources:** [The Hacker News](https://thehackernews.com/2026/05/lazarus-deploys-remotepe-memory-only.html), [Bitdefender](https://www.bitdefender.com/en-gb/blog/hotforsecurity/lazarus-group-accused-of-stealing-11-million-in-crypto-from-taiwanese-exchange-bitopro), [Rewterz](https://rewterz.com/threat-advisory/lazarus-group-deploys-multi-stage-remotepe-malware-in-stealth-financial-attacks-active-iocs)

### Shai-Hulud / Hades Variant — PyPI Bioinformatics Packages (TeamPCP)

- **Attack type:** Compromised maintainer accounts / malicious update / Python import hook injection
- **Ecosystem:** PyPI
- **Package(s) affected:** 19 bioinformatics packages including Dynamo, Spateo, CoolBox, U-FISH, and Napari-UFISH (37 malicious wheels published); exact version counts per package not yet fully enumerated by responders
- **Impact:** Python `.pth` import hook (`litellm_init.pth` or `router_runtime.js` in PyPI variant) executes on every Python interpreter startup — not merely on package import. Payload steals GitHub PATs, PyPI credentials, AWS/GCP/Azure/Kubernetes secrets, SSH keys, npm tokens, and AI assistant configs. Triple-redundant exfiltration channel. Linux persistence via systemd user service at `~/.config/sysmon/sysmon.py`.
- **IOCs:** `router_runtime.js` (PyPI payload file), `litellm_init.pth` (Python site-packages import hook), systemd service `~/.config/sysmon/sysmon.py`, C2 domain `checkmarx.zone` (persistence polling every 50 minutes), `filev2.getsession.org` (P2P C2 channel)
- **Mitigation:** Immediately audit `site-packages` for `.pth` files not part of known packages. Remove any package installed from the affected bioinformatics namespace during the June 8 window. Rotate all credentials accessible to Python processes on affected systems — GitHub tokens, PyPI tokens, cloud keys. Regenerate lockfiles and verify tarball-vs-source integrity. Confirm no systemd user services were added under `~/.config/sysmon/`.
- **Source:** [BleepingComputer](https://www.bleepingcomputer.com/news/security/new-shai-hulud-attack-trojanizes-19-science-focused-pypi-packages/), [SecurityWeek](https://www.securityweek.com/over-100-npm-pypi-packages-hit-in-new-shai-hulud-supply-chain-attacks/), [phoenix.security IOC analysis](https://phoenix.security/sha1-hulud-shai-hulud-worm-analysis-persistence-iocs/)

### Shai-Hulud / Miasma Variant — @redhat-cloud-services npm Namespace

- **Attack type:** Compromised maintainer account / CI/CD pipeline injection / preinstall hook
- **Ecosystem:** npm
- **Package(s) affected:** 32 packages published under the `@redhat-cloud-services` npm namespace; combined 116,991 weekly downloads
- **Impact:** `preinstall` hook delivers the Miasma payload. Sweeps AWS SSM parameters and Secrets Manager entries across all regions via full SigV4 credential resolution (env vars, INI profiles, IRSA OIDC, ECS metadata, EC2 IMDS). Also targets GCP, Azure, and Kubernetes credentials. Payload encrypts harvested data with operator RSA public key before exfil to `git-tanstack.com`.
- **IOCs:** `router_init.js` (npm payload filename), `@tanstack/setup` optional dependency pinned to attacker commit `d446803f`, C2 `git-tanstack.com`, secondary exfil `audit.checkmarx.cx`
- **Mitigation:** Pin all `@redhat-cloud-services` packages to versions published before June 1, 2026 and verify tarball integrity against published checksums. Rotate all cloud credentials on any CI/CD runner that installed affected packages. Audit internal Artifactory/Nexus caches — cached malicious versions persist after upstream cleanup. Check npm lockfiles for `@tanstack/setup` optional dependency pinned to commit `d446803f`.
- **Source:** [NHS England Digital Advisory](https://digital.nhs.uk/cyber-alerts/2026/cc-4781), [phoenix.security IOC analysis](https://phoenix.security/sha1-hulud-shai-hulud-worm-analysis-persistence-iocs/)
