> **Daily Brief.** The current threat landscape is defined by a convergence of high-severity network-edge exploits, sustained Chinese and Iranian state-sponsored intrusion campaigns, and an aggressive wave of developer supply chain compromises. Citrix NetScaler continues to see large-scale active exploitation of CVE-2026-3055 (CVSS 9.8) by opportunistic actors and likely state-aligned groups targeting SAML authentication infrastructure. The Miasma npm supply chain attack—a direct evolution of the Shai-Hulud campaign—has compromised over 32 Red Hat @redhat-cloud-services packages, exfiltrating multi-cloud credentials at scale. Iranian APT Screening Serpens is sustaining espionage campaigns against U.S., Israeli, and UAE aerospace and telecom sectors using evolved DLL sideloading and AppDomainManager hijacking. Defenders should prioritize patching Citrix, Android, and Windows Netlogon exposures while auditing CI/CD pipelines for Miasma persistence artifacts.

## Executive Summary

- **CVE-2026-3055** (Citrix NetScaler, CVSS 9.8): Large-scale exploitation confirmed by Fortinet against internet-facing SAML IDP appliances; patch immediately to 14.1-66.59 / 13.1-62.23 or newer.
- **CVE-2025-48595** (Android Framework integer overflow, CVSS 8.4): Actively exploited in targeted attacks; added to CISA KEV June 2, 2026 with federal deadline June 5.
- **Miasma npm supply chain attack**: 32+ @redhat-cloud-services packages backdoored to harvest GitHub Actions tokens, AWS/Azure/GCP credentials, SSH keys, Kubernetes configs, and .env files; self-propagating worm with developer-tool persistence.
- **Salt Typhoon** (Chinese APT) remains active inside U.S. networks following a confirmed fresh penetration into Congressional email systems; operations expanded to Norway and Singapore in February 2026.
- **Screening Serpens** (Iranian APT): Six new RAT variants (MiniUpdate, MiniJunk V2) deployed in February–April 2026 against U.S., Israeli, and UAE aerospace/defense/telecom via spear-phishing and AppDomainManager hijacking.
- **June 2026 Patch Tuesday** (June 9): CVE-2026-41096 (Windows DNS RCE, CVSS 9.8), CVE-2026-42826 (Azure DevOps info disclosure, CVSS 10.0), CVE-2026-40361 (Outlook zero-click RCE), and CVE-2026-41089 (Windows Netlogon stack overflow, CVSS 9.8) are among the highest-priority fixes.

## Software Supply Chain Attacks

> Incidents, campaigns, and vulnerabilities affecting software build pipelines, open-source packages, CI/CD infrastructure, third-party libraries, and code-signing infrastructure — reported within the last 7 days.

### @redhat-cloud-services npm Packages — Miasma Campaign
- **Attack type:** Compromised maintainer account / malicious commit bypassing code review
- **Ecosystem:** npm
- **Package(s) affected:** 32 packages under @redhat-cloud-services namespace including @redhat-cloud-services/vulnerabilities-client, @redhat-cloud-services/rbac-client, @redhat-cloud-services/sources-client, @redhat-cloud-services/remediations-client, @redhat-cloud-services/topological-inventory-client, @redhat-cloud-services/tsc-transform-imports, @redhat-cloud-services/rule-components — 96 affected versions; 116,000+ accumulated downloads
- **Impact:** Preinstall lifecycle hook executes 4.2 MB obfuscated JavaScript payload at `npm install` time, sweeping GitHub Actions tokens, AWS/GCP/Azure credentials, HashiCorp Vault tokens, Kubernetes kubeconfig/service account tokens, npm/PyPI publish tokens, SSH private keys, Docker registry credentials, GPG keys, and all .env files. Malware also attempts privilege escalation via container bind-mount of /etc/sudoers.d, establishes persistence via Claude Code SessionStart hooks (~/.claude/settings.json) and VSCode tasks.json ("runOn": "folderOpen"), and avoids execution on Russian-language systems. Uniquely encrypted payload per infection impedes hash-based detection.
- **IOCs:** Exfiltration staged to attacker-controlled GitHub repositories with description "Miasma: The Spreading Blight"; C2 masquerading as `api.anthropic[.]com:443/v1/api`; commit messages containing `IfYouInvalidateThisTokenItWillNukeTheComputerOfTheOwner:<token>`; persistence artifacts in `~/.claude/settings.json`, `.vscode/tasks.json`, `.github/workflows/codeql.yml`, `.github/setup.js`
- **Mitigation:** Immediately isolate affected hosts; remove compromised package versions; rotate ALL credentials that were present on the machine (cloud, npm, SSH, Vault, Kubernetes); audit GitHub Actions secrets and workflow run history; search for persistence artifacts listed above; treat uninstalling the npm package as insufficient — full credential rotation is required. See [Socket remediation guide](https://socket.dev/blog/mini-shai-hulud-campaign-hits-red-hat-cloud-services-npm-packages).
- **Source:** [The Hacker News](https://thehackernews.com/2026/06/miasma-supply-chain-attack-compromises.html), [Wiz](https://www.wiz.io/blog/miasma-supply-chain-attack-targeting-redhat-npm-packages), [Microsoft Security Blog](https://www.microsoft.com/en-us/security/blog/2026/06/02/preinstall-persistence-inside-red-hat-npm-miasma-credential-stealing-campaign/)

### Shai-Hulud / Mini Shai-Hulud — Ongoing npm / PyPI Campaign
- **Attack type:** CI/CD pipeline injection / stolen credentials / SLSA provenance abuse
- **Ecosystem:** npm, PyPI
- **Package(s) affected:** 170+ packages across npm (@tanstack/* and others) and PyPI; affected projects include Mistral AI, OpenSearch Project, TanStack; 518M+ combined monthly downloads
- **Impact:** Install-time credential harvesting targeting developer cloud credentials and SSH keys; self-propagating worm via GitHub API; first confirmed npm packages with valid SLSA provenance signatures used for malicious artifact propagation
- **IOCs:** Packages published by `TeamPCP` / `TGR-CRI-1135` accounts; GitHub repositories with SLSA-signed but malicious artifacts
- **Mitigation:** Audit dependencies against published compromise lists from Ox Security and Socket.dev; regenerate lock files from known-good versions; review GitHub Actions for unauthorized workflow commits; revoke any publish tokens present in compromised environments
- **Source:** [Ox Security](https://www.ox.security/blog/shai-hulud-here-we-go-again-170-packages-hit-across-npm-pypi/), [Unit 42](https://unit42.paloaltonetworks.com/monitoring-npm-supply-chain-attacks/)
