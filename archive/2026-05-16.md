## Executive Summary

- **CVE-2026-42897 (Microsoft Exchange OWA, CVSS 8.1) — Active exploitation; CISA KEV; no patch.** Stored XSS / spoofing in Exchange Server SE / 2019 / 2016 triggered when a victim opens a crafted email in OWA. Microsoft has shipped only a mitigation through the Exchange Emergency Mitigation Service (EM2.1.x); a code fix is still in development. Exchange Online is **not** affected. Verify EM is auto-applying and audit OWA logs back to May 12.
- **CVE-2026-20182 (Cisco Catalyst SD-WAN Controller / Manager, CVSS 10.0) — Active exploitation by UAT-8616; CISA KEV; FCEB deadline May 17 (tomorrow).** Auth bypass in the `vdaemon` peering service (DTLS/UDP 12346) → log in as `vmanage-admin` non-root, then pivot to NETCONF and SSH key injection. Same actor and same surface as CVE-2026-20127. Apply Cisco's fixed releases per `cisco-sa-sdwan-rpa2-v69WY2SW`; restrict UDP 12346 and TCP 22/830 to administrative networks today.
- **CVE-2026-7482 "Bleeding Llama" (Ollama, CVSS 9.1) — Unauthenticated heap OOB read; ~300k servers exposed.** Crafted GGUF posted to `/api/create` triggers an out-of-bounds heap read in the GGUF tensor-shape loader; data is then exfiltrated by pushing the resulting model artifact to an attacker registry via `/api/push`. Leaks env vars, API keys, system prompts, and concurrent users' chat data. Patch to **Ollama 0.17.1**; never expose `/api/create` to the internet.
- **CVE-2026-44112 / 44113 / 44115 / 44118 "Claw Chain" (OpenClaw, ≤ 2026.4.22) — Sandbox-escape → cred theft → priv-esc → persistence.** Cyera's chain across two TOCTOU race conditions in the OpenShell sandbox, a credential-exposure flaw, and a runtime privilege escalation. Shodan + ZoomEye see ~245k exposed instances. Upgrade to **OpenClaw 2026.4.22**; treat any agent runtime reachable from untrusted prompt input as in-scope.
- **CVE-2026-44338 (PraisonAI ≤ 4.6.33) — Missing-auth in legacy Flask API; weaponised within 4 hours of disclosure.** `AUTH_ENABLED = False` hard-coded in `src/praisonai/api_server.py` lets any reachable caller hit `/agents` and `/chat` and execute the configured agents.yaml workflow. Sysdig observed `CVE-Detector/1.0` scanning vulnerable endpoints 3h44m after disclosure. Upgrade to **4.6.34**.
- **CVE-2026-31431 "Copy Fail" (Linux kernel) — KEV deadline expired May 15.** Container hosts and multi-tenant cloud workloads remain the highest-risk surface; any unpatched kernel today should be treated as critical-priority remediation.
- **The Gentlemen RaaS — Group's own back-end leaked May 4; affiliate model and tooling now public.** 332 published victims in the first five months of 2026 (≈ #2 globally per Check Point); initial access predominantly via CVE-2024-55591 FortiOS auth bypass, SystemBC SOCKS5 tunneling for stealth, cross-platform Go encryptors. Anonymous group exfiltrated chat logs, ransom transcripts, and tooling discussions — high-value reading for IR teams who have ever responded to a Gentlemen incident.
- **PathWiper (Russia-nexus APT) — Destructive wiper against Ukrainian critical-infrastructure entity (Talos).** Delivered through a legitimate endpoint-management console; chain is `*.bat` → `uacinstall.vbs` → `sha256sum.exe`. NTFS-aware wipe (MBR / $MFT / $LogFile / $Boot, dismounts volumes first) including network drives. Operationally similar to HermeticWiper-class wipers but with cleaner volume enumeration; Talos has published Snort rules and IOCs.

## APT / Threat Actor Activity

### UAT-8616 (Cisco Talos cluster; assessed highly sophisticated; long-running)

- **Targets:** Organisations running internet-reachable Cisco Catalyst SD-WAN fabrics (Controller / Manager / vSmart / vManage) — predominantly large enterprise and managed-service operator deployments. Censys still observed ~600 SD-WAN Manager instances exposed in May, with ~25% additionally exposing TCP 22 or 830.
- **TTPs:** Exploit of `vdaemon` peering DTLS (UDP 12346) — historically CVE-2026-20127, now also CVE-2026-20182 (`T1190`). Optional deliberate version downgrade to re-expose CVE-2022-20775 for LPE to root (`T1574`). Post-compromise: log-in as `vmanage-admin`, write unauthorised SSH keys to `/home/vmanage-admin/.ssh/authorized_keys/`, set `PermitRootLogin`, create/delete malicious user accounts, manipulate NETCONF config (`T1098`, `T1078.003`, `T1136`, `T1070`), and clear `auth.log`, `bash_history`, `cli-history` to cover tracks.
- **Recent activity:** Talos formally disclosed CVE-2026-20182 exploitation on May 14, 2026, with attribution to UAT-8616 at high confidence; the same cluster has been active since at least 2023. CISA added CVE-2026-20182 to KEV on May 14 with a May 17 federal deadline.
- **Attribution confidence:** **High** — Talos correlation of intrusion artefacts, infrastructure overlap, and identical post-compromise behavioural pattern across CVE-2026-20127 and CVE-2026-20182 victims.
- **Sources:** [Talos — UAT-8616 SD-WAN exploitation](https://blog.talosintelligence.com/uat-8616-sd-wan/) · [Talos — ongoing exploitation](https://blog.talosintelligence.com/sd-wan-ongoing-exploitation/) · [The Hacker News — CISA KEV add](https://thehackernews.com/2026/05/cisa-adds-cisco-sd-wan-cve-2026-20182.html) · [Help Net Security](https://www.helpnetsecurity.com/2026/05/15/cisco-sd-wan-zero-day-cve-2026-20182/)

### The Gentlemen RaaS (active since mid-2025; #2 globally by victim count in 2026)

- **Targets:** Cross-industry; victim concentration in the United States, United Kingdom, Germany, Australia, and Romania per Check Point's analysis of their leaked SystemBC C2 backend. Affiliate-driven RaaS; multi-OS (Windows + Linux Go encryptors).
- **TTPs:** Predominantly initial access via **CVE-2024-55591** (FortiOS / FortiProxy auth bypass) — operators maintain a 14,700-device list of pre-exploited FortiGates and 969 brute-forced VPN credentials. Post-access: SystemBC for SOCKS5 proxy tunneling and stealth (`T1090.002`), RPC-based Cobalt Strike deployment to remote hosts (`T1021.003`), domain-level GPO/policy abuse for fan-out ransomware deployment (`T1484.001`), and Go-based cross-platform lockers. SystemBC C2 traffic is RC4-encrypted custom protocol — distinctive on the wire. April DFIR Report observation also chained EtherRAT (Ethereum smart-contract C2 resolution via `1rpc.io`) and the AI-generated TukTuk framework into a Gentlemen deployment, exfiltrating to cloud SaaS prior to encryption.
- **Recent activity:** 332 published victims in Jan–May 2026 (Group-IB / Check Point). On or just before **May 4, 2026** an anonymous group breached The Gentlemen's own backend and exfiltrated internal chat logs, organisational rosters, ransom negotiation transcripts, and tooling discussions — a portion of which Check Point captured before the data was taken down.
- **Attribution confidence:** **High** for the RaaS-program-level attribution (their own leaked infrastructure); affiliate-level attribution remains opaque by design.
- **Sources:** [Check Point Research — DFIR The Gentlemen](https://research.checkpoint.com/2026/dfir-report-the-gentlemen/) · [Check Point — leak overview](https://blog.checkpoint.com/research/when-the-ransomware-gang-gets-hacked-what-the-gentlemen-leak-reveals-about-modern-ransomware-risk) · [BleepingComputer — SystemBC + Gentlemen](https://www.bleepingcomputer.com/news/security/the-gentlemen-ransomware-now-uses-systembc-for-bot-powered-attacks/) · [The DFIR Report — Flash Alert](https://thedfirreport.com/2026/05/11/flash-alert-etherrat-and-tuktuk-c2-end-in-the-gentleman-ransomware/) · [Group-IB — Hasta la vista, Hastalamuerte](https://www.group-ib.com/blog/hastalamuerte-gentlemen-raas-ttps/)

### PathWiper operators (Russia-nexus APT; Talos high confidence)

- **Targets:** Ukrainian critical-infrastructure entity (specific sector not publicly disclosed). Talos assesses the attack as a high-confidence Russia-nexus operation based on TTP overlap and wiper capabilities consistent with prior destructive Ukrainian-targeted operations (HermeticWiper / IsaacWiper lineage).
- **TTPs:** **Hands-on-keyboard via compromised legitimate endpoint-administration framework** — the attackers had access to the admin console and used it to push commands to managed endpoints (`T1078`, `T1072`). Execution chain: BAT (`T1059.003`) → `uacinstall.vbs` (`T1059.005`) dropped to Windows TEMP → `sha256sum.exe` payload (`T1036.005`, masquerading as a benign hashing utility). The wiper enumerates **all** drives — local, network (queries `HKEY_USERS\Network\<drive_letter>|RemovePath`), and dismounted — verifies volume labels and valid records, dismounts targets, then overwrites MBR, $MFT, $LogFile, $Boot and file content with random data (`T1485`, `T1561.002`).
- **Recent activity:** Disclosed and analysed by Cisco Talos; this is the second uniquely Ukrainian-targeted wiper of 2026. The use of the legitimate admin framework rather than a custom dropper is the operationally significant change — defenders cannot rely on signatured droppers and must instead instrument the admin console itself.
- **Attribution confidence:** **High** (Talos) — TTP and capability overlap with prior Russia-nexus Ukrainian wipers; deployment method consistent with known operator tradecraft.
- **Sources:** [Cisco Talos — PathWiper writeup](https://blog.talosintelligence.com/pathwiper-targets-ukraine/) · [BleepingComputer](https://www.bleepingcomputer.com/news/security/new-pathwiper-data-wiper-malware-hits-critical-infrastructure-in-ukraine/) · [SecurityWeek](https://www.securityweek.com/destructive-pathwiper-targeting-ukraines-critical-infrastructure/) · [Dark Reading](https://www.darkreading.com/cyberattacks-data-breaches/pathwiper-attack-critical-infrastructure-ukraine)

### Lazarus Group ("Mach-O Man" / Hidden Cobra / APT38; DPRK RGB)

- **Targets:** Fintech and crypto executives, developers, and decision-makers operating macOS; Quetzal Team / ANY.RUN observed targeting of crypto-exchange and DeFi infrastructure staff via compromised Telegram contacts.
- **TTPs:** ClickFix via Telegram lure (`T1566.003`), user-paste-and-execute terminal command (`T1204.002`), `codesign` ad-hoc signing of fake Zoom/Teams/Meet app bundles (`T1553.002`), Go/Mach-O four-stage chain — `teamsSDK.bin` stager → `D1*.bin` profiler (`T1082`, `T1057`) → `minst2.bin` persistence via `~/Library/LaunchAgents/com.onedrive.launcher.plist` (`T1543.001`) → `macrasv2` stealer harvesting browser SQLite, browser extensions, and macOS Keychain entries (`T1555.003`, `T1539`). Exfiltration via Telegram Bot API (`T1567.002`); C2 over HTTP using Go default User-Agent on ports 8888 / 9999.
- **Recent activity:** Quetzal Team / ANY.RUN published the full IOC set and technical writeup April 21, 2026; the kit remains in active distribution and DPRK's cumulative 2026 crypto haul remains > $577M (Drift Protocol $285M, KelpDAO $293M, etc.). Operators left the Telegram bot token exposed inside `macrasv2`, which has aided attribution and takedown but does not mitigate already-deployed implants.
- **Attribution confidence:** **High** — TTP overlap with prior ClickFix and PyLangGhost RAT campaigns publicly attributed to Lazarus; `geniex-client` code lineage matches prior Quetzal Lazarus tracking.
- **Sources:** [ANY.RUN — Mach-O Man writeup](https://any.run/cybersecurity-blog/lazarus-macos-malware-mach-o-man/) · [Quetzal Team original](https://open.substack.com/pub/quetzalteam/p/north-koreas-safari-hunting-for-rats) · [Cybersecurity News — Mach-O Man](https://cybersecuritynews.com/mach-o-man-macos-malware-lazarus/)
