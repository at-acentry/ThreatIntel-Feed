> **Daily Brief.** The final days of May 2026 arrive with a dense threat landscape dominated by two converging storylines: an unpatched Microsoft Exchange Server zero-day (CVE-2026-42897) that requires only a crafted email to hijack OWA sessions, and the widening fallout from the TeamPCP/Mini Shai-Hulud supply chain worm that spread credential-stealing payloads through 160+ npm and PyPI packages. Iranian APT group Screening Serpens has significantly escalated its espionage tempo against U.S., Israeli, and UAE aerospace and defense targets, deploying six new RAT variants. Simultaneously, the TrapDoor campaign continues to poison the crypto and AI developer ecosystems across npm, PyPI, and Crates.io. Organizations running on-premises Exchange, any npm or PyPI-heavy build pipelines, or internet-exposed cloud infrastructure should treat today's recommended actions as urgent.

## Executive Summary

- **CVE-2026-42897 (Exchange OWA XSS, CVSS 8.1):** Actively exploited zero-day requires only a crafted email to execute JavaScript in victim OWA sessions; no patch available as of 2026-05-30; CISA KEV-listed with federal deadline May 29.
- **TeamPCP / Mini Shai-Hulud supply chain worm** compromised 160+ npm and PyPI packages including TanStack, Mistral AI, and UiPath; self-propagating payload harvests cloud credentials (GitHub, npm, AWS, GCP, Azure, K8s) and re-publishes poisoned versions.
- **TrapDoor campaign** targets crypto/DeFi/AI developers across npm, PyPI, and Crates.io with 34+ malicious packages; uses GitHub Gists for C2 exfiltration; earliest activity May 22.
- **Screening Serpens (UNC1549/Smoke Sandstorm)** escalated espionage against U.S., Israeli, and UAE aerospace/defense/telco; deployed six new RAT variants (MiniUpdate, MiniJunk V2) using DLL sideloading + AppDomainManager hijacking.
- **Fancy Bear (APT28)** continues global campaign leveraging Windows zero-day CVE-2026-21513 and Office bug CVE-2026-21509 with steganography-based C2 and integrated wiper capability.
- **PCPJack worm** spreading across exposed Docker, Kubernetes, Redis, and MongoDB infrastructure, exploiting five CVEs; uses Telegram for C2 and evicts competing TeamPCP infections.

## Software Supply Chain Attacks

> Incidents, campaigns, and vulnerabilities affecting software build pipelines, open-source packages, CI/CD infrastructure, third-party libraries, and code-signing infrastructure — reported within the last 7 days.

### TanStack / Mini Shai-Hulud (TeamPCP) — npm + PyPI Worm

- **Attack type:** CI/CD pipeline injection via GitHub Actions cache poisoning + "Pwn Request" (pull_request_target abuse)
- **Ecosystem:** npm, PyPI
- **Package(s) affected:** 84 malicious versions across 42 `@tanstack/*` packages (19:20–19:26 UTC, May 11, 2026); expanded to 160+ packages total including Mistral AI, UiPath, OpenSearch, Guardrails AI; 404 malicious versions published
- **Impact:** Lifecycle hook execution steals GitHub tokens, npm tokens, AWS credentials (IMDSv2), GCP/Azure credentials, Kubernetes service account tokens, HashiCorp Vault tokens, and environment variables; self-propagates by re-publishing poisoned versions of any package the victim has publish access to; installs persistence daemon (macOS: `~/Library/LaunchAgents/com.user.gh-token-monitor.plist`; Linux: `~/.config/systemd/user/gh-token-monitor.service`)
- **IOCs:** All `@tanstack/*` versions published May 11–12, 2026; check `npm audit` and lock files for affected version ranges
- **Mitigation:** Upgrade to clean `@tanstack/*` versions published after May 12, 2026 by official maintainers; remove persistence daemon; rotate all credentials from any affected machine; audit CI/CD logs for `pull_request_target` workflow triggers from forks
- **Source:** [TanStack Postmortem](https://tanstack.com/blog/npm-supply-chain-compromise-postmortem), [Orca Security](https://orca.security/resources/blog/tanstack-npm-supply-chain-worm/), [StepSecurity](https://www.stepsecurity.io/blog/mini-shai-hulud-is-back-a-self-spreading-supply-chain-attack-hits-the-npm-ecosystem)

### TrapDoor — npm, PyPI, Crates.io (Crypto/AI Developer Targeting)

- **Attack type:** Coordinated malicious package publication from cluster of adversary-controlled accounts
- **Ecosystem:** npm, PyPI, Crates.io
- **Package(s) affected:** 34+ malicious packages across 384 versions; representative samples: `git-config-sync` (PyPI), `token-usage-tracker` (npm), `sui-framework-helpers` (Crates.io); targets crypto, DeFi, Solana, and AI developer communities
- **Impact:** Credential theft (crypto wallets, API keys, SSH keys); uses XOR encryption with hardcoded key `cargo-build-helper-2026` (Crates.io samples); npm samples use Fernet + ECDH encryption; exfiltration via GitHub Gists (`ddjidd564.github.io` C2 domain)
- **IOCs:** C2 domain: `ddjidd564.github.io`; campaign marker: `P-2024-001`; earliest malicious activity: 2026-05-22 20:20 UTC
- **Mitigation:** Audit dependency trees for the three named packages and any unfamiliar packages targeting crypto/Solana/AI namespaces; block `ddjidd564.github.io` at DNS layer; verify package maintainer identity before installation
- **Source:** [The Hacker News](https://thehackernews.com/2026/05/trapdoor-supply-chain-attack-spreads.html), [Socket.dev](https://socket.dev/blog/trapdoor-crypto-stealer-npm-pypi-crates), [SlowMist](https://slowmist.medium.com/threat-intelligence-trapdoor-analysis-a-cross-ecosystem-supply-chain-credential-theft-operation-a9a4e11616ea)
