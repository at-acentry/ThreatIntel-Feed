## APT / Threat Actor Activity

### APT28 / Fancy Bear (aliases: Sofacy, Sednit, STRONTIUM, UAC-0001)
- **Targets:** Security and defense organizations in Ukraine; expanding to Western logistics and technology companies.
- **TTPs:** Spear-phishing with ZIP archives disguised as "Anhang.pdf.zip" containing PyInstaller-packaged executables (`T1566.001`, `T1204.002`); LAMEHUG malware embeds an LLM (Alibaba Qwen 2.5-Coder via Hugging Face API) to dynamically generate Windows recon command chains (`systeminfo`, `wmic`, `tasklist`, `netstat`, `ipconfig`, `dsquery` — `T1059.003`, `T1082`); blends C2 traffic with legitimate Hugging Face API requests for evasion (`T1102`).
- **Recent activity:** CERT-UA tracked ~270 Hugging Face tokens used for LAMEHUG authentication; observed C2 endpoint `stayathomeclasses[.]com/slpw/up[.]php`; at least two additional payload variants identified ("AI_generator_uncensored_Canvas_PRO_v0.9.exe", "image.py").
- **Attribution confidence:** High — CERT-UA and multiple vendors (Cato Networks, Recorded Future-affiliated researchers) link the campaign to GRU Unit 26165.
- **Sources:** [CERT-UA via The Hacker News](https://thehackernews.com/2025/07/cert-ua-discovers-lamehug-malware.html), [Cato Networks](https://www.catonetworks.com/blog/cato-ctrl-threat-research-analyzing-lamehug/)

### Silent Ransom Group (aliases: UNC3753, Luna Moth, Chatty Spider)
- **Targets:** U.S. law firms, financial and professional services; dozens of organizations targeted January–May 2026.
- **TTPs:** Invoice-themed phishing emails from consumer accounts with no malicious payload, designed solely to prime victims for a follow-up call (`T1566`); vishing impersonating internal IT helpdesk (`T1598.004`); convinces victims to join remote sessions via Teams/Zoom/Quick Assist and install RMM tools (AnyDesk, Zoho Assist, Bomgar, SuperOps — `T1219`); most recently, in-person physical intrusion; exfiltration over web services (`T1567`).
- **Recent activity:** FBI advisory confirms escalation to physical intrusion against U.S. law firms; ransom demands reported as high as $20 million.
- **Attribution confidence:** High — consistent tradecraft tracked by FBI, Halcyon, and multiple incident responders since 2022.
- **Sources:** [Halcyon](https://www.halcyon.ai/ransomware-alerts/an-old-tactic-returns-silent-ransom-groups-active-use-of-physical-intrusion-against-u-s-law-firms), [BleepingComputer](https://www.bleepingcomputer.com/news/security/silent-ransom-group-targets-law-firms-with-fake-it-support-calls/)

## Software Supply Chain Attacks

> Incidents, campaigns, and vulnerabilities affecting software build pipelines, open-source packages, CI/CD infrastructure, third-party libraries, and code-signing infrastructure — reported within the last 7 days.

### IronWorm (npm)
- **Attack type:** Compromised maintainer account distributing a Rust-based information stealer via trojanized packages.
- **Ecosystem:** npm / GitHub Actions.
- **Package(s) affected:** Packages published from compromised npm account "asteroiddao" (linked to GitHub org `asteroid-dao`, member account `ocrybit`); malicious commits authored under spoofed identity "claude" (`claude@users.noreply.github.com`) across nine GitHub organizations; 37 packages affected per JFrog.
- **Impact:** Scrapes 86 environment variables and credential files for OpenAI Codex, Anthropic Claude, Google Gemini, Cursor, AWS, Docker, Kubernetes, npm, vault configs, and Exodus crypto wallets (skips the attacker's own wallet, currently empty); hides behind an eBPF kernel rootkit (defeated by kernel lockdown); communicates over Tor; swaps GitHub Actions workflows to exfiltrate secrets as build artifacts (no external C2 needed); abuses npm Trusted Publishing to mint short-lived tokens for republishing poisoned versions.
- **IOCs:** Compromised npm account `asteroiddao`; GitHub org `asteroid-dao`; GitHub member `ocrybit`; commit author `claude@users.noreply.github.com`; attacker wallet `0x7e28D9889f414B06c19a22A9Bd316f0AC279a4d6` (empty, monitor for activity).
- **Mitigation:** Audit recent commits authored as "claude@users.noreply.github.com"; rotate AWS/Docker/K8s/npm/AI-tool credentials on any system running affected packages; disable eBPF-capable kernel features where not required, or enable kernel lockdown; enable GitHub Actions workflow-change alerting; review Trusted Publishing token issuance logs.
- **Source:** [JFrog via The Hacker News](https://thehackernews.com/2026/06/ironworm-and-new-miasma-worm-variant.html)

### Miasma worm variant — "Phantom Gyp" (npm)
- **Attack type:** Self-propagating credential-harvesting worm distributed via a novel `binding.gyp`-triggered execution technique, derivative of the Shai-Hulud worm.
- **Ecosystem:** npm (cross-platform: Linux, macOS, Windows; Linux CI/CD runners primarily targeted).
- **Package(s) affected:** 57 npm packages across 286+ malicious versions, including `ai-sdk-ollama`, `autotel`, `awaitly`, `effect-analyzer`, `eslint-plugin-awaitly`, `executable-stories-cypress`, `http-uploader-dev`, `mountly`, `node-env-resolver`, `node-env-resolver-aws`. Follows the original Miasma incident (32 packages, 90+ versions, @redhat-cloud-services namespace), root-caused by Red Hat to a compromised GitHub account.
- **Impact:** Abuses a 157-byte `binding.gyp` file to trigger code execution during `npm install`, bypassing pre/postinstall script monitoring ("Phantom Gyp" technique); downloads the Bun JavaScript runtime per-platform to run a credential harvester targeting AWS, GCP, Azure, HashiCorp Vault, Docker, Kubernetes, GitHub Actions, npm, RubyGems, PyPI, SSH, password managers, and AI coding assistants; injects persistent backdoor files into repos that execute when a developer opens the project in an AI-assisted IDE; in CI/CD, scrapes GitHub Actions runner memory, escalates via passwordless sudo, and republishes poisoned packages with forged SLSA provenance. A follow-on stage searches GitHub for commits containing the string "firedalazer" (successor to "FIRESCALE") to fetch an additional Shai-Hulud-derived payload, turning GitHub into an adaptive C2/dead-drop channel.
- **IOCs:** Exfiltration GitHub account `liuende501` (236 staged repos, now inaccessible); dead-drop search string `firedalazer`; exfil repos carrying description "Miasma : The Spreading Blight" on accounts `0tabek16` and `windy629` (82+ repos as of disclosure).
- **Mitigation:** Rotate all credentials on systems that installed an affected version; disable install scripts and native module rebuilds (`npm config set ignore-scripts true`) by default; pin dependencies with integrity hashes; audit AI-assisted IDE project files for unexpected backdoor scripts; monitor for outbound GitHub API traffic from CI/CD runners as a potential C2 indicator.
- **Source:** [StepSecurity / Endor Labs / Microsoft / OX Security via The Hacker News](https://thehackernews.com/2026/06/ironworm-and-new-miasma-worm-variant.html)
