> **Daily Brief.** The week ending June 12 saw the threat landscape shaped by three concurrent pressures: an actively exploited Check Point VPN zero-day (CVE-2026-50751) linked to Qilin ransomware deployments, a self-propagating supply chain worm (Shai-Hulud/Miasma/Hades) that compromised over 100 npm and PyPI packages including Red Hat's Hybrid Cloud Console SDK, and a surge of World Cup-themed fraud infrastructure ahead of the FIFA 2026 kickoff on June 11. CISA added three new vulnerabilities to its KEV catalog on June 9 — including a Cisco SD-WAN zero-day and an Arista EOS flaw for which no patch is planned — raising the immediate remediation burden for enterprise IT teams. Chinese APT operators Salt Typhoon and Volt Typhoon remain active in telecommunications and critical infrastructure environments respectively, while North Korea's TraderTraitor group continues pivoting between cryptocurrency theft and ransomware deployment. Organizations running internet-facing network edge devices should treat this week as a high-urgency patching window.

## Executive Summary

- **Check Point VPN zero-day (CVE-2026-50751, CVSS 9.3)** exploited since May 7 by a Qilin ransomware affiliate using IKEv1 authentication bypass; Check Point has released a hotfix but attacks are ongoing and no prior authentication is required.
- **CISA KEV additions (June 9)** include CVE-2026-11645 (Chrome V8 OOB, exploited in wild), CVE-2026-7473 (Arista EOS tunnel flaw, no patch available), and CVE-2026-20245 (Cisco SD-WAN Manager local privilege-escalation to root, no patch, no workaround); federal remediation deadline June 23.
- **Shai-Hulud supply chain worm** variants Miasma (npm) and Hades (PyPI) have compromised 57+ npm packages and 29+ PyPI packages respectively; the worm is self-propagating — it harvests credentials from the victim environment and uses them to re-infect packages the victim has publish access to.
- **FIFA World Cup 2026 fraud surge**: The FBI has issued a PSA listing 35+ confirmed fake FIFA domains; Group-IB tracks 4,300+ fraudulent FIFA domains with one Chinese-speaking group ("Ghost Stadium") operating 300+ near-perfect clones; Android banking trojans (Massiv, Perseus) hidden in pirate streaming apps are targeting fans in host cities.
- **Salt Typhoon** continues expanding beyond U.S. telecoms into South American networks, deploying new implants (TernDoor, PeerTime, BruteEntry); FBI confirmed in February 2026 that activity remains active and ongoing.
- **SolarWinds Serv-U DoS (CVE-2026-28318)** added to KEV June 5; unauthenticated attackers can crash Serv-U via a crafted `Content-Encoding: deflate` POST request — patch to Serv-U 15.5.4 Hotfix 1 is available and must be prioritized.

**CVE-2026-50751 notes:** Authentication bypass in Check Point Remote Access VPN using deprecated IKEv1 protocol. Unauthenticated, remote exploitation requires no user interaction. Exploitation began May 7, with a surge in early June. Check Point confirmed post-exploitation ELF payload retrieval from attacker-controlled VPS infrastructure (Kaupo Cloud HK, Shock Hosting, Vultr). Hotfix available; disable IKEv1 and enforce IKEv2 with mandatory machine certificate authentication.

**CVE-2026-20245 notes:** No patch, no workaround. Attackers are chaining this with previously disclosed CVE-2026-20182 and CVE-2026-20127 for initial netadmin access before escalating to root. Network isolation of SD-WAN Manager is the only available control.

**CVE-2026-7473 notes:** Arista has stated no patches will be developed for this flaw, citing risk of breaking existing configurations. Apply ACLs to upstream devices to permit only legitimate tunnel source IPs.

## Software Supply Chain Attacks

> Incidents, campaigns, and vulnerabilities affecting software build pipelines, open-source packages, CI/CD infrastructure, third-party libraries, and code-signing infrastructure — reported within the last 7 days.

### Shai-Hulud Worm — Miasma Variant (npm)

- **Attack type:** Compromised maintainer account / CI/CD pipeline injection / self-propagating worm
- **Ecosystem:** npm
- **Package(s) affected:** 57+ packages including `@redhat-cloud-services/*` (32 packages, 96 versions on June 1), Vapi server SDK, `ai-sdk-ollama`, `autotel`, `awaitly`, `executable-stories`, `node-env-resolver`, `wrangler-deploy`; 300+ malicious package versions total identified by June 5
- **Impact:** Multi-stage credential-harvesting dropper executed at `npm install` via weaponized `binding.gyp` (bypasses postinstall hook detection). Scans local filesystem and connected cloud services for credentials, API keys, and tokens. Uses harvested credentials to republish malicious versions to packages the victim has publish access to, enabling exponential spread. Exfiltrates to C2 endpoint `t.m-kosche.com` disguised as OpenTelemetry trace telemetry. Also creates public GitHub repositories under victim accounts and commits exfiltrated data as JSON files.
- **IOCs:** C2 endpoint: `t.m-kosche.com`; presence of `bun` binary at unexpected temp path (`/tmp/b-<random>/bun` on Linux or `/var/folders/<…>/b-<random>/bun` on macOS); temp file matching `/tmp/p<base36>.js` (both artifacts self-delete on successful execution). Hash-based IOCs are per-infection due to unique payload encryption — pull current hashes from Snyk Advisory DB and Sonatype OSS Index.
- **Mitigation:** Audit `binding.gyp` files in all installed npm packages; rotate all CI/CD credentials, cloud API keys, and npm publish tokens for any developer who installed affected packages; re-lock `package-lock.json` from a clean environment; monitor for unexpected `bun` process execution and outbound connections to `t.m-kosche.com` on port 443
- **Source:** [SecurityWeek](https://www.securityweek.com/over-100-npm-pypi-packages-hit-in-new-shai-hulud-supply-chain-attacks/), [Harness Blog](https://www.harness.io/blog/shai-hulud-miasma-inside-the-compromise-of-red-hats-packages), [JFrog Research](https://research.jfrog.com/post/shai-hulud-miasma-redhat-cloud-services/)

### Shai-Hulud Worm — Hades Variant (PyPI)

- **Attack type:** Compromised maintainer account / malicious update / startup hook injection
- **Ecosystem:** PyPI
- **Package(s) affected:** 29+ packages across two waves (June 7 and June 8); bioinformatics, graph machine learning, and MCP-themed packages specifically targeted in second wave; at least 471 total malicious artifacts (npm + PyPI combined)
- **Impact:** Injects `-setup.pth` files that execute at every Python interpreter startup (not just on import), fetching the Bun JavaScript runtime and executing JavaScript-based credential stealer. "Phantom releases" pushed to PyPI without corresponding GitHub commits to evade detection. Hades variant uses a loader/payload split across `sys.path` to frustrate hash-based detections. Same data exfiltration model as Miasma (GitHub repository dead-drop).
- **IOCs:** Presence of unexpected `.pth` files in Python site-packages directory; `bun` runtime binary appearing in temp paths; outbound connections from Python interpreter processes to external IPs/domains. Specific per-package hashes available via Socket.dev and EndorLabs advisories.
- **Mitigation:** Audit `site-packages` for unexpected `.pth` files; run `pip list --outdated` and cross-reference against Socket.dev / PyPI security advisories; regenerate lockfiles and rotate all secrets accessible from affected CI/CD environments; consider enforcing `PYTHONPATH` restrictions in production pipelines
- **Source:** [SecurityWeek](https://www.securityweek.com/over-100-npm-pypi-packages-hit-in-new-shai-hulud-supply-chain-attacks/), [Socket.dev](https://socket.dev/blog/shai-hulud-descends-to-hades-miasma-pypi-wave), [EndorLabs](https://www.endorlabs.com/learn/shai-hulud-hades-wave-hits-six-pypi-bioinformatics-packages)

- [ ] **SOC:** Hunt for outbound connections to `t.m-kosche.com` and `cloudflariz.com` from developer machines, CI/CD runners, and build agents. Enable detections for `bun` binary execution from temp directories. Query EDR for `Content-Encoding: deflate` POST requests to Serv-U listening ports. Add FIFA phishing domain list above to DNS sinkhole or proxy blocklist. Alert on LummaC2, Vidar, and RedLine stealer C2 callbacks — IOCs available via ThreatFox.
- [ ] **IT / Patch Mgmt:** **(Priority 1)** Patch Check Point Remote Access VPN — disable IKEv1, enforce IKEv2 with mandatory machine certificate authentication, apply hotfix; FCEB deadline was immediate. **(Priority 2)** Update Chrome to 149.0.7827.102+ (CVE-2026-11645) across all endpoints. **(Priority 3)** Patch SolarWinds Serv-U to 15.5.4 Hotfix 1 or block POST requests containing `content-encoding` header at perimeter; FCEB deadline June 19. **(Priority 4)** For Cisco Catalyst SD-WAN Manager (CVE-2026-20245) — no patch exists; isolate management interface, restrict access to trusted admin hosts only. **(Priority 5)** For Arista EOS (CVE-2026-7473) — no patch planned; apply upstream ACLs to restrict tunnel source IPs.
- [ ] **Dev / AppSec:** Immediately audit all npm and PyPI dependencies installed after May 25 for Shai-Hulud/Miasma/Hades compromise. Rotate all CI/CD secrets, npm publish tokens, cloud API keys, and GitHub tokens on developer machines and build systems. Inspect `binding.gyp` files and Python `site-packages` `.pth` files for unexpected content. Regenerate `package-lock.json` and `requirements.txt` lockfiles from a verified clean baseline. Consider adding `npm audit` and Socket.dev scanning to pre-commit hooks.
- [ ] **Leadership:** The Shai-Hulud worm's self-propagating model means a single compromised developer's machine can cascade into downstream customer supply chains — assess exposure of any packages or SDKs published by your organization. Qilin's exploitation of an unpatched VPN zero-day for over a month before discovery underscores the need for network segmentation and VPN access logging. Advise any staff attending FIFA World Cup venues to avoid unofficial streaming apps and public Wi-Fi for work applications; issue guidance on FIFA ticket purchasing from official channels only.
