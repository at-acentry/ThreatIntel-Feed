> **Daily Brief.** June 11 marks both the opening of the FIFA World Cup 2026 and Microsoft's largest-ever Patch Tuesday — a confluence that threat actors are actively exploiting. Patch Tuesday delivered 198 CVEs including three publicly disclosed zero-days, while Google patched a Chrome V8 zero-day (CVE-2026-11645) that is actively exploited in the wild and KEV-listed. Cisco's Catalyst SD-WAN Manager carries a critical zero-day (CVE-2026-20245) with no patch yet available. **CVE-2026-3300 context:** Unauthenticated RCE in Everest Forms Pro (4,000+ WordPress installs). Attackers create rogue admin account "diksimarina" via /wp-admin/admin-ajax.php. Patch to ≥ 1.9.13 released March 18, 2026.

**CVE-2026-11645 context:** Out-of-bounds read/write in Chrome's V8 JavaScript engine exploitable via crafted HTML page. Fixed in Chrome 149.0.7827.102/.103. Fifth Chrome zero-day exploited in 2026.

**CVE-2026-20245 context:** CLI injection in Cisco SD-WAN Manager requiring netadmin privileges. No patch available as of June 11. Chaining with prior SD-WAN CVEs (CVE-2026-20182, CVE-2026-20127) can reduce access prerequisites.

**CVE-2026-23111 context:** A single removed `!` inverted an activation condition in the nf_tables abort path, causing a use-after-free reachable via unprivileged user namespaces (enabled by default on Ubuntu, Debian, Fedora). Public exploit code released by Exodus Intelligence on June 8.

### Shai-Hulud / Miasma / Hades Worm Campaign (TeamPCP)

- **Attack type:** Compromised maintainer accounts + CI/CD pipeline injection + self-propagating worm via OIDC trust abuse
- **Ecosystem:** npm + PyPI
- **Packages affected:**
  - **npm (Miasma wave, June 1–5):** 32 packages under @redhat-cloud-services scope (Red Hat GitHub repositories compromised); 57+ npm packages total confirmed malicious across the campaign
  - **PyPI (Hades wave, June 8–10):** `bramin`, `cmd2func`, `coolbox`, `dynamo-release`, `executor-engine`, `executor-http`, `funcdesc`, `magique`, `magique-ai`, `mrbios`, `napari-ufish`, `nucbox`, `okite`, `pantheon-agents`, `pantheon-toolsets`, `spateo-release`, `synago`, `ufish`, `uprobe` (19 packages, 37 malicious wheel files); secondary bioinformatics cluster: `embiggen`, `ensmallen`, `gpsea`, `mflux-streamlit`, `nhmpy`, `ppkt2synergy`, `pyphetools`
- **Impact:** Credential theft at Python interpreter startup via `.pth` file execution before package import; harvests GitHub tokens, npm/PyPI/RubyGems tokens, AWS/GCP/Azure/K8s credentials, SSH keys, CI/CD secrets, `.env` files, shell history; lateral spread via SSH/SCP and OIDC trust abuse; GitHub org secrets extracted via Actions runners; AI tool backdooring (Claude, Codex, Gemini, Copilot, Cline, Aider); wiper daemon `gh-token-monitor` destroys data if stolen GitHub token is revoked
- **Novel evasion:** Prompt-injection text embedded in package code instructs LLM-based security scanners to classify package as safe; decoy traffic sent to Anthropic API to confuse network analysis; Russian locale check to avoid self-infection
- **IOCs:** GitHub repos with description "Hades - The End for the Damned" or "Hades * The End for the Damned"; `*-setup.pth` files in PyPI package distributions; payload file `_index.js`; Bun JavaScript runtime download from GitHub
- **Mitigation:** Immediately uninstall affected packages; rotate ALL CI/CD secrets (GitHub, npm, PyPI, cloud providers); audit Python virtualenvs for unexpected `.pth` files (`find . -name "*-setup.pth"`); regenerate lockfiles and verify checksums; disable OIDC trust on compromised repositories; update pip and npm to latest versions
- **Source:** [The Hacker News](https://thehackernews.com/2026/06/hades-pypi-attack-19-packages-poisoned.html), [SecurityWeek](https://www.securityweek.com/over-100-npm-pypi-packages-hit-in-new-shai-hulud-supply-chain-attacks/), [StepSecurity](https://www.stepsecurity.io/blog/the-hades-campaign-pypi-packages), [Socket.dev](https://socket.dev/blog/shai-hulud-descends-to-hades-miasma-pypi-wave), [NHS England](https://digital.nhs.uk/cyber-alerts/2026/cc-4781)
