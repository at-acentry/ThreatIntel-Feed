## Software Supply Chain Attacks

> Incidents, campaigns, and vulnerabilities affecting software build pipelines, open-source packages, CI/CD infrastructure, third-party libraries, and code-signing infrastructure — reported within the last 7 days.

### @redhat-cloud-services npm Packages (Miasma / Mini Shai-Hulud)

- **Attack type:** Compromised maintainer account → malicious orphan CI/CD commit → trusted publishing abuse
- **Ecosystem:** npm
- **Package(s) affected:** 32 packages under `@redhat-cloud-services/*` namespace; 96 versions; ~116,991 cumulative weekly downloads. Confirmed compromised packages include `@redhat-cloud-services/types` v3.6.1 and at least 31 others
- **Impact:** Self-propagating credential-stealing worm (Miasma, a variant of Mini Shai-Hulud). Sweeps for: HashiCorp Vault tokens, Kubernetes kubeconfig/service account tokens, npm and PyPI publish tokens, SSH private keys, Docker registry credentials, GPG keys, `.env` files. Exfiltrates via three channels including GitHub repository commits to attacker-controlled repos and abuse of `api.anthropic.com` endpoint for Anthropic API key validation. Worm propagates by using stolen npm tokens to re-infect downstream packages
- **IOCs:** Malicious packages: `@redhat-cloud-services/types@3.6.1`; File hash (known malicious loader): `b86c5ae9e95bd841a595440faa3eb6317441e746f241ae8fd641ab59ed1d1966`; Attacker-created GitHub repos carrying description string `"Miasma: The Spreading Blight"` (hunting signal); Exfil path pattern: `results/<timestamp>-<counter>.json` in compromised victim GitHub accounts
- **Mitigation:** Immediately audit npm install logs for any `@redhat-cloud-services/*` install between May 31–June 3, 2026. Run `npm audit` and pin to post-remediation versions published after June 3, 2026. Rotate all credentials present in developer environments that ran affected packages: Vault tokens, kubeconfig, SSH keys, Docker credentials, npm/PyPI tokens. Regenerate lock files from known-clean state. Review GitHub repo commit history for orphan `ci.yaml` / `_index.js` patterns
- **Source:** [Wiz Blog](https://www.wiz.io/blog/miasma-supply-chain-attack-targeting-redhat-npm-packages), [Microsoft Security Blog](https://www.microsoft.com/en-us/security/blog/2026/06/02/preinstall-persistence-inside-red-hat-npm-miasma-credential-stealing-campaign/), [JFrog Security](https://research.jfrog.com/post/shai-hulud-miasma-redhat-cloud-services/), [Snyk](https://snyk.io/blog/miasma-supply-chain-attack-malicious-code-redhat-cloud-services-npm-packages/)

### TeamPCP / Tanstack Package Campaign

- **Attack type:** Coordinated malicious package publication (typosquatting / namespace hijack)
- **Ecosystem:** npm and PyPI
- **Package(s) affected:** 84 malicious artifacts across 42 `@tanstack/*` packages published within a 6-minute window on May 11, 2026
- **Impact:** Credential harvesting targeting developer cloud credentials and SSH keys; part of a broader coordinated wave that hit npm, PyPI, and Docker Hub within 48 hours
- **IOCs:** `@tanstack/*` packages published by "TeamPCP" account between May 11 05:00–05:06 UTC; exact package list in GitGuardian reporting
- **Mitigation:** Verify publisher identity for all `@tanstack/*` dependencies; pin to known-good versions pre-dating May 11, 2026; regenerate credentials present in affected CI/CD environments
- **Source:** [GitGuardian](https://blog.gitguardian.com/four-credential-harvesting-campaigns-hit-open-source-ecosystems-in-two-weeks/), [MSN/Tech](https://www.msn.com/en-us/news/technology/three-separate-supply-chain-attacks-hit-npm-pypi-and-docker-hub-within-48-hours-all-three-targeted-developer-cloud-credentials-and-ssh-keys/ar-AA23zEJz)
