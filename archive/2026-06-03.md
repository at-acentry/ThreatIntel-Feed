> **Daily Brief.** The Cisco Catalyst SD-WAN exploitation campaign attributed to UAT-8616 and at least ten distinct threat clusters continues to widen in scope, with CISA adding CVE-2026-20182 (CVSS 10.0) to its KEV catalog and patch deadlines now passed for federal agencies. Iranian APT activity under the Screening Serpens umbrella has intensified with at least six new RAT variants deployed since February, and the DriveSurge initial access broker continues to compromise thousands of legitimate websites to deliver ClickFix and FakeUpdate payloads at scale.

- Iranian Screening Serpens deployed six new RAT variants (MiniUpdate, MiniJunk v2) targeting U.S., Israeli, and UAE technology sector personnel via spear-phishing recruitment lures.

## APT / Threat Actor Activity

### UAT-8616 (aliases: linked to broader Chinese-nexus ORB network cluster)
- **Targets:** Enterprise SD-WAN infrastructure globally; network service providers, financial institutions
- **TTPs:** Authentication bypass via Cisco SD-WAN CVEs → SSH key injection, NETCONF modification, root escalation; deploys Godzilla/Behinder/XenShell webshells, Sliver C2, AdaptixC2, KScan + Nim-based RAT, XMRig miners, gsocket tunneling (`T1190`, `T1505.003`, `T1078`, `T1059.004`, `T1071.001`)
- **Recent activity:** Active ITW exploitation of CVE-2026-20182 from May 14, 2026; prior exploitation of CVE-2026-20127; infrastructure overlaps with known ORB (Operational Relay Box) networks tracked by Cisco Talos. At least ten distinct post-compromise activity clusters documented.
- **Attribution confidence:** High (Cisco Talos) — consistent TTPs with prior UAT-8616 activity
- **Sources:** [Cisco Talos](https://blog.talosintelligence.com/sd-wan-ongoing-exploitation/), [THN](https://thehackernews.com/2026/05/cisa-adds-cisco-sd-wan-cve-2026-20182.html)

### Salt Typhoon / Volt Typhoon (aliases: GhostEmperor, RedMike, UNC5807, OPERATOR PANDA)
- **Targets:** Telecommunications carriers, ISPs, government agencies, critical infrastructure (water, energy) — primarily U.S. and allied nations
- **TTPs:** Living-off-the-land on routers and network edge devices, long-dwell persistence, authorized wiretap interception data collection; Volt Typhoon focuses on pre-conflict positioning for disruption (`T1190`, `T1133`, `T1563`, `T1016`, `T1082`)
- **Recent activity:** CISA issued a supplementary advisory (February 2026) noting intensified Volt Typhoon activity in water and communications sectors; Salt Typhoon still present inside multiple U.S. telecom networks; a separate Chinese campaign (February 2026) impacted 50+ telecoms and government agencies across 42 countries.
- **Attribution confidence:** High (CISA, FBI, NSA joint advisory; multiple vendor corroboration)
- **Sources:** [CISA AA25-239A](https://www.cisa.gov/news-events/cybersecurity-advisories/aa25-239a), [Eclypsium](https://eclypsium.com/blog/the-rise-of-chinese-apt-campaigns-volt-typhoon-salt-typhoon-flax-typhoon-and-velvet-ant/)

### Screening Serpens / MuddyWater (aliases: Boggy Serpens, Seedworm, Static Kitten, Mango Sandstorm, MERCURY)
- **Targets:** Technology sector professionals in the U.S., Israel, UAE, and broader Middle East
- **TTPs:** Spear-phishing via personalized recruitment lures impersonating hiring platforms → DLL sideloading → MiniUpdate and MiniJunk v2 RATs; C2 infrastructure hosted on Azure, 3–5 unique domains per target; prior use of Chaos ransomware false-flag for deniability (`T1566.001`, `T1574.002`, `T1102`, `T1059.003`)
- **Recent activity:** Six new RAT variants deployed between February and April 2026 against U.S., Israeli, and UAE targets. MiniUpdate and MiniJunk v2 families identified by Unit 42. C2 domain `screenai.online` (registered via NameCheap August 2025) used in earlier campaigns.
- **Attribution confidence:** High (Unit 42 Palo Alto, SOCRadar) — attributed to Iranian MOIS
- **Sources:** [Unit 42](https://unit42.paloaltonetworks.com/tracking-iran-apt-screening-serpens/), [SecurityWeek](https://www.securityweek.com/iranian-apt-intrusion-masquerades-as-chaos-ransomware-attack/)

### DriveSurge (Initial Access Broker — unattributed)
- **Targets:** Broad opportunistic targeting of website visitors; macOS users specifically targeted in payload stage; downstream victims vary by payload operator
- **TTPs:** Compromise legitimate websites → inject zTDS traffic distribution system → redirect visitors to ClickFix social engineering pages or FakeUpdate browser impersonation pages → deliver macOS infostealer or Windows malware (`T1189`, `T1204.002`, `T1059.004`, `T1566.002`)
- **Recent activity:** Active since at least September 2025; thousands of compromised websites identified by Silent Push as of May/June 2026; operates as Pay-Per-Install IAB. Payload C2 confirmed at 147.45.42.205:8133. macOS clipboard hijacking via fake reCAPTCHA delivers multi-stage shell command downloading infostealer from 46.226.166.57.
- **Attribution confidence:** Low — likely Russian-language operator (Russian-language ADS panel, NiceNIC registrar, zTDS open-source Russian TDS)
- **Sources:** [Silent Push](https://www.silentpush.com/blog/drivesurge/), [BleepingComputer](https://www.bleepingcomputer.com/news/security/hackers-hijack-thousands-of-sites-for-clickfix-and-fakeupdate-attacks/)

## Software Supply Chain Attacks

> Incidents, campaigns, and vulnerabilities affecting software build pipelines, open-source packages, CI/CD infrastructure, third-party libraries, and code-signing infrastructure — reported within the last 7 days.

### Red Hat @redhat-cloud-services npm Packages — Miasma
- **Attack type:** Compromised maintainer account / GitHub Actions OIDC token abuse / CI/CD pipeline injection
- **Ecosystem:** npm
- **Package(s) affected:** 32 packages under `@redhat-cloud-services` scope; 96 compromised versions; ~116,000 cumulative downloads. Key packages include `@redhat-cloud-services/frontend-components@7.7.2–7.7.5`, `rbac-client@9.0.3–9.0.6`, `insights-client@4.0.4–4.0.7`, and 29 others (full list in Wiz advisory).
- **Impact:** Malicious preinstall lifecycle hook executes obfuscated 4.2 MB JavaScript payload during `npm install`. Sweeps GitHub Actions tokens, AWS keys (AccessKeyId, SecretAccessKey, session tokens), GCP application default credentials and all service account identities, Azure service principal and managed identity tokens, HashiCorp Vault tokens, Kubernetes service accounts, kubeconfig files, npm/PyPI publish tokens, SSH private keys, Docker registry credentials, GPG keys, and all `.env` files. Generates uniquely encrypted payload per infection to defeat hash-based detection.
- **IOCs:** Repository description string `Miasma: The Spreading Blight`; user-agent `google-api-nodejs-client/7.0.0 gl-node/20.11.0 gccl/7.0.0` (GCP credential sweep); malicious orphan commit SHAs: `8bf051251ec3b973e39a313547e53421a2f8d2f6`, `608d01124cd6b5b8c55888e984b4c4d9b06fa686`, `ab9903d9edc720d1e11ea7d3d3e7a1c456f44ff7`, `7569d69cf3684a792ce63d19b6e0d9d192597963`
- **Mitigation:** Most malicious versions revoked as of 1 June 2026 afternoon. Immediately audit developer workstations, CI/CD pipelines, and GitHub Actions for exposure. Rotate all GitHub tokens, SSH keys, cloud credentials, and CI/CD secrets on any system that ran affected package versions. Pin known-good versions from `package-lock.json`; verify against Red Hat's RHSB-2026-006 advisory. Implement `npm audit` and SBOM generation.
- **Source:** [Wiz](https://www.wiz.io/blog/miasma-supply-chain-attack-targeting-redhat-npm-packages), [Red Hat RHSB-2026-006](https://access.redhat.com/security/vulnerabilities/RHSB-2026-006), [Orca Security](https://orca.security/resources/blog/red-hat-npm-supply-chain-attack/)

## Recommended Actions

- [ ] **SOC:** Hunt for Cisco SD-WAN webshell filenames (`20251117022131.jsp`, `conf.jsp`, `sysv.jsp`, `sysinit.jsp`, `vmurnp_ikp.jsp`); enable Snort SIDs 66482–66483 (CVE-2026-20182) and 66461–66462/66468–66469 (CVE-2026-20133/20128/20122); block all Talos Cluster IPs listed above; alert on `Unix.Backdoor.NimPlant-10059957`, `Unix.Tool.GSocket-10059956`, and related ClamAV signatures
- [ ] **SOC:** Hunt for DriveSurge injection pattern: external JS resources matching filename `t.[12hexchars].js` or `ext-b.[12hexchars].js`; alert on outbound connections to 147.45.42.205:8133 and 46.226.166.57
- [ ] **SOC:** Search for npm package install logs referencing `@redhat-cloud-services` packages with versions listed in Wiz RHSB-2026-006; look for repo descriptions matching "Miasma: The Spreading Blight"
- [ ] **IT / Patch Mgmt:** Priority order — (1) Cisco SD-WAN Controller/Manager: patch CVE-2026-20182 immediately (CISA deadline passed); (2) Palo Alto PAN-OS: patch CVE-2026-0257 by 19 June and CVE-2026-0300 ASAP; (3) MOVEit Automation: upgrade to 2025.1.5, 2025.0.9, or 2024.1.8 (CVE-2026-4670); (4) Android devices: apply June 2026 security update (CVE-2025-48595 deadline 5 June); (5) Linux servers: apply kernel updates for Dirty Frag (CVE-2026-43284, CVE-2026-43500)
- [ ] **Dev / AppSec:** Immediately audit any use of `@redhat-cloud-services` npm packages — check lock files against RHSB-2026-006 safe version list; rotate all CI/CD secrets, cloud credentials, and npm tokens on affected build systems; implement GitHub Actions OIDC token scope restrictions and require branch protection rules to prevent orphan commit injection
- [ ] **Leadership:** The Miasma supply chain attack directly impacts Red Hat enterprise tooling used across hybrid cloud environments — client communications may be warranted for organizations running Insights, RBAC, or frontend-components packages. The Cisco SD-WAN exploitation chain represents systemic risk to network management plane integrity; customers with SD-WAN deployments should be asked to confirm patch status.
