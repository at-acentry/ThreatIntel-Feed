> **Daily Brief.** The month opens with a dense threat landscape: the Linux kernel "Copy Fail" privilege escalation (CVE-2026-31431) remains actively exploited in cloud and container environments days after its KEV addition, while two Microsoft Defender zero-days patched last week are already confirmed exploited in live intrusions. The TanStack/Mini Shai-Hulud and TrapDoor supply chain campaigns from late May continue to pose residual risk to developer environments, with stolen CI secrets and cloud credentials circulating in attacker infrastructure. Iranian APT Screening Serpens (UNC1549) is sustaining a targeted espionage push against aerospace, defense, and telecoms organizations across the U.S., Israel, and UAE using freshly developed RAT variants — organizations in those sectors should treat any unsolicited job-lure email as high-priority IOC triage.

## Executive Summary

- **CVE-2026-31431 (Linux "Copy Fail")** — KEV-listed LPE in all Linux kernels since 2017; trivial PoC public; exploitation confirmed in cloud and containerized workloads. Patch to kernel 6.18.22+ immediately.
- **Microsoft Defender zero-days (CVE-2026-41091 / CVE-2026-45498)** — elevation-of-privilege and denial-of-service flaws actively exploited via RedSun and UnDefend tooling; FCEB patch deadline June 3, 2026.
- **NGINX Rift (CVE-2026-42945)** — 18-year-old heap overflow (CVSS 9.2) in `ngx_http_rewrite_module`; exploitation attempts observed in the wild since May 16; unauthenticated RCE possible on misconfigured servers.
- **TanStack/Mini Shai-Hulud supply chain worm** — TeamPCP compromised 160+ npm/PyPI packages via poisoned GitHub Actions cache; credential exfiltration to `getsession.org` infrastructure; OpenAI confirmed two employee devices impacted.
- **TrapDoor campaign** — 34 malicious packages across npm, PyPI, and Crates.io targeting crypto/DeFi/AI developers; first activity May 22; credential-stealing payloads with cron/systemd persistence.
- **Screening Serpens (UNC1549)** — Iran-nexus APT deploying MiniUpdate and MiniJunk V2 RATs via job-lure spear-phishing against U.S., Israeli, and Emirati defense-industrial targets.

## Software Supply Chain Attacks

> Incidents, campaigns, and vulnerabilities affecting software build pipelines, open-source packages, CI/CD infrastructure, third-party libraries, and code-signing infrastructure — reported within the last 7 days.

### TanStack / Mini Shai-Hulud (TeamPCP)

- **Attack type:** CI/CD pipeline injection — poisoned GitHub Actions pnpm cache via malicious `pull_request_target` workflow
- **Ecosystem:** npm, PyPI
- **Package(s) affected:** `@tanstack/router` and 160+ downstream dependent packages; confirmed impact on Mistral AI and UiPath packages
- **Impact:** Exfiltrates GitHub tokens, npm tokens, AWS/GCP/Azure/Kubernetes credentials, HashiCorp Vault tokens, and all environment variables; installs persistence daemon; potentially wipes developer home directories via destructive daemon
- **IOCs:** C2 typosquat domain `git-tanstack.com`; exfiltration endpoint `filev2.getsession.org`; seed nodes `seed1.getsession.org`, `seed2.getsession.org`, `seed3.getsession.org`; C2 IP `83.142.209.194`; persistence at `~/Library/LaunchAgents/com.user.gh-token-monitor.plist` (macOS) or `~/.config/systemd/user/gh-token-monitor.service` (Linux)
- **Mitigation:** Pin all `@tanstack/*` packages to verified clean versions per [TanStack postmortem](https://tanstack.com/blog/npm-supply-chain-compromise-postmortem); regenerate all CI secrets, cloud credentials, and npm tokens from any system that installed affected packages; audit GitHub Actions workflow files for unauthorized modifications; block `git-tanstack.com` and `*.getsession.org` at DNS/proxy layer
- **Source:** [Orca Security](https://orca.security/resources/blog/tanstack-npm-supply-chain-worm/), [StepSecurity](https://www.stepsecurity.io/blog/mini-shai-hulud-is-back-a-self-spreading-supply-chain-attack-hits-the-npm-ecosystem)

### TrapDoor Campaign (TeamPCP)

- **Attack type:** Typosquatting and malicious package publication across multiple ecosystems
- **Ecosystem:** npm, PyPI, Crates.io
- **Package(s) affected:** 34 malicious packages, 384+ versions; packages impersonating generic developer tools and security scanners targeting crypto/DeFi/AI developers; published from npm account `asdxzxc`, PyPI accounts `asdmini67` and `dae5411`
- **Impact:** Credential theft (crypto keys, AWS/GitHub tokens), persistence via cron/systemd/Git hooks, lateral movement via SSH; Python packages execute on import; Rust packages use `build.rs` hooks; AI assistant poisoning via hidden `.cursorrules`/`CLAUDE.md` instructions
- **IOCs:** Payload delivery domain `ddjidd564.github.io`; npm account `asdxzxc`; PyPI accounts `asdmini67`, `dae5411`; internal campaign marker `P-2024-001`; payload script `trap-core.js`
- **Mitigation:** Audit `package.json` and `pyproject.toml` for packages from listed accounts; run `npm audit` and scan with Socket.dev; regenerate any secrets on affected developer machines; lock dependency versions and enable registry provenance verification
- **Source:** [Socket.dev](https://socket.dev/blog/trapdoor-crypto-stealer-npm-pypi-crates), [THN](https://thehackernews.com/2026/05/trapdoor-supply-chain-attack-spreads.html)

### Megalodon — GitHub Actions Mass Backdoor (TeamPCP)

- **Attack type:** CI/CD pipeline injection — mass GitHub Actions workflow poisoning
- **Ecosystem:** GitHub Actions
- **Package(s) affected:** 5,561 public GitHub repositories; 5,718 malicious commits pushed over 6 hours on May 18, 2026
- **Impact:** Exfiltrates CI secrets, cloud credentials, SSH keys, OIDC tokens, and source code to attacker C2 at `216.126.225.129:8443`; two workflow variants: SysDiag (mass, triggers on every push/PR) and Optimize-Build (targeted dormant backdoor activatable via GitHub API)
- **IOCs:** C2 IP `216.126.225.129` port `8443`; forged author identities `build-bot`, `auto-ci`, `ci-bot`, `pipeline-bot`; forged emails `build-system@noreply.dev`, `ci-bot@automated.dev`; commit window May 18, 2026, 11:36–17:48 UTC
- **Mitigation:** Audit all `.github/workflows/` files for unrecognized additions since May 18; revoke and rotate CI secrets, OIDC tokens, and cloud credentials from affected repos; enable branch protection and require signed commits; review GitHub audit logs for API-triggered workflow dispatches
- **Source:** [StepSecurity](https://www.stepsecurity.io/blog/megalodon-mass-github-actions-secret-exfiltration-across-5-500-public-repositories), [SecurityWeek](https://www.securityweek.com/over-5500-github-repositories-infected-in-megalodon-supply-chain-attack/)
