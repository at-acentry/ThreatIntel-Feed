> **Daily Brief.** Today's collection window surfaces three distinct threat tracks demanding immediate attention: CISA added three new KEV entries yesterday (2026-05-27) confirming active supply-chain exploitation of DAEMON Tools, TanStack, and the Nx Console developer extension. State-sponsored cluster CL-STA-1132 continues to exploit the critical PAN-OS zero-day CVE-2026-0300 for root-level RCE on PA-Series and VM-Series firewalls, now confirmed active since April 9. The cross-ecosystem "TrapDoor" campaign — spreading credential-stealing malware across npm, PyPI, and Crates.io — has posted 384 malicious package versions since May 22 and is actively targeting crypto, DeFi, and AI developer communities. A second zero-day in Microsoft Exchange Server (CVE-2026-42897) remains unpatched and actively exploited via crafted OWA emails. Defenders should prioritize PAN-OS patching, audit developer environments for TrapDoor-linked packages, and ensure Exchange Emergency Mitigation Services are enabled.

## Executive Summary

- **KEV additions (2026-05-27):** CISA catalogued three supply-chain CVEs — CVE-2026-8398 (DAEMON Tools), CVE-2026-45321 (TanStack/npm), and CVE-2026-48027 (Nx Console VSCode extension) — all confirmed actively exploited for credential harvesting.
- **PAN-OS RCE (CVE-2026-0300, CVSS 9.3):** State-sponsored cluster CL-STA-1132 has been exploiting this buffer overflow in the User-ID Authentication Portal since April 9; post-exploitation activity includes AD enumeration, log destruction, and EarthWorm/ReverseSocks5 tunneling.
- **Exchange zero-day (CVE-2026-42897, CVSS 8.1):** No permanent patch available; XSS via crafted email enables arbitrary JavaScript in OWA; Exchange Online is not affected, but all on-prem Exchange 2016/2019/SE installations are.
- **TrapDoor supply-chain campaign:** 34 malicious packages across npm, PyPI, and Crates.io target developer secrets (GitHub tokens, CI/CD credentials, cloud API keys) and poison AI assistant config files (`.cursorrules`, `CLAUDE.md`) to trigger auto-exfiltration.
- **Fortinet FortiClient EMS (CVE-2026-35616, CVSS 9.8):** Pre-auth API bypass under active exploitation since March 31; known to deliver EKZ infostealer disguised as a Fortinet patch.
- **Iranian APT Screening Serpens (UNC1549):** Six new RAT variants (MiniUpdate, MiniJunk V2) deployed in an escalating espionage campaign since February targeting US, Israeli, and UAE tech and defense sectors.

## APT / Threat Actor Activity

### CL-STA-1132 (aliases: unassigned — likely state-nexus, attribution pending)

- **Targets:** Network perimeter infrastructure globally; confirmed victims include PA-Series and VM-Series firewall operators across multiple verticals
- **TTPs:** Unauthenticated buffer overflow exploitation (`T1190`), shellcode injection into nginx worker processes (`T1055`), EarthWorm/ReverseSocks5 tunneling (`T1572`, `T1090`), Active Directory enumeration via compromised service account (`T1018`, `T1087`), aggressive log destruction (`T1070`)
- **Recent activity:** Exploitation of CVE-2026-0300 confirmed active since April 9, 2026; attackers achieved RCE approximately one week after initial probing; post-exploitation involved AD recon and systematic crash-log deletion to impair forensic investigation
- **Attribution confidence:** Medium — Palo Alto Unit 42 designates as likely state-sponsored; geopolitical indicators consistent with East Asia nexus, no public definitive attribution
- **Sources:** [Unit 42](https://unit42.paloaltonetworks.com/captive-portal-zero-day/), [Rapid7 ETR](https://www.rapid7.com/blog/post/etr-critical-buffer-overflow-in-palo-alto-networks-pan-os-user-id-authentication-portal-cve-2026-0300/), [Help Net Security](https://www.helpnetsecurity.com/2026/05/06/palo-alto-firewalls-vulnerability-exploited-cve-2026-0300/)

- **Sources:** [Unit 42 / Screening Serpens 2026](https://unit42.paloaltonetworks.com/tracking-iran-apt-screening-serpens/), [The Jerusalem Post](https://www.jpost.com/middle-east/iran-news/article-897154)

## Software Supply Chain Attacks

> Incidents, campaigns, and vulnerabilities affecting software build pipelines, open-source packages, CI/CD infrastructure, third-party libraries, and code-signing infrastructure — reported within the last 7 days.

### TrapDoor — Cross-Ecosystem Credential Harvester (npm / PyPI / Crates.io)

- **Attack type:** Typosquatting and compromised maintainer-style publishing; AI assistant config poisoning
- **Ecosystem:** npm, PyPI, Crates.io
- **Package(s) affected:** `prompt-engineering-toolkit`, `solidity-deploy-guard`, `defi-threat-scanner`, `dev-env-bootstrapper`, `wallet-security-checker` (npm); `eth-security-auditor` (PyPI); `move-analyzer-build` (Crates.io) — 34 packages total, 384 malicious versions; earliest observed: `eth-security-auditor@0.1.0` published 2026-05-22 20:20 UTC
- **Impact:** On import/install, packages harvest GitHub and npm tokens, CI/CD secrets, cloud API keys, SSH credentials; achieve persistence via cron jobs, systemd services, and developer workflow hooks; poison `.cursorrules` and `CLAUDE.md` with hidden instructions to trick AI coding assistants into triggering credential exfiltration during "security scans"
- **IOCs:** GitHub/npm account `ddjidd564` / `asdxzxc`; C2 staging at `ddjidd564.github.io/defi-security-best-practices/`; internal marker `P-2024-001`; XOR key `cargo-build-helper-2026` (Crates.io); npm postinstall hook deploys `trap-core.js` (1,149-line harvester)
- **Mitigation:** Uninstall all listed packages immediately; audit `package-lock.json`, `Pipfile.lock`, and `Cargo.lock` for the above package names and any version published after 2026-05-22; revoke and rotate all GitHub tokens, npm tokens, and cloud credentials in affected development environments; purge any `.cursorrules` or `CLAUDE.md` not authored by your team
- **Source:** [Socket.dev](https://socket.dev/blog/trapdoor-crypto-stealer-npm-pypi-crates), [The Hacker News](https://thehackernews.com/2026/05/trapdoor-supply-chain-attack-spreads.html)

### Mini Shai-Hulud / TeamPCP — TanStack and 170+ npm Package Compromise (CVE-2026-45321)

- **Attack type:** GitHub Actions cache poisoning via pull_request_target workflow abuse; compromised CI/CD pipeline injection
- **Ecosystem:** npm, PyPI (secondary)
- **Package(s) affected:** `@tanstack/router` and 170+ downstream npm packages including Mistral AI, UiPath, OpenSearch (1.3M weekly downloads), Guardrails AI; 404 malicious versions published in two waves (April 29 and May 11, 2026)
- **Impact:** Malicious versions harvest GitHub and npm tokens, CI/CD secrets, and cloud credentials from build environments; 15 million+ users potentially exposed through downstream dependency trees
- **IOCs:** Attacker forked TanStack/router repository; malicious pnpm store injected into GitHub Actions cache; check GitHub Actions workflow files for unexpected `pull_request_target` triggers with forked repo references
- **Mitigation:** Pin `@tanstack/router` and all affected packages to versions released before April 29, 2026 or the latest clean release; regenerate all CI/CD secrets and GitHub PATs; audit GitHub Actions workflow permissions; run `npm audit` and review lockfiles
- **Source:** [SafeDep](https://safedep.io/mass-npm-supply-chain-attack-tanstack-mistral/), [Tenable FAQ](https://www.tenable.com/blog/mini-shai-hulud-frequently-asked-questions)

### Nx Console (CVE-2026-48027) and DAEMON Tools (CVE-2026-8398) — CISA KEV (2026-05-27)

- **Attack type:** Compromised extension/software distribution — malicious versions published to trusted distribution channels
- **Ecosystem:** VS Code Marketplace (Nx Console); software distribution (DAEMON Tools Lite)
- **Package(s) affected:** Nx Console VS Code extension (malicious version); DAEMON Tools Lite (affected version unspecified)
- **Impact:** Nx Console: compromised extension fetched obfuscated payload harvesting credentials from disk and memory; DAEMON Tools: unspecified high-impact confidentiality/integrity/availability compromise
- **IOCs:** Malicious Nx Console version details pending vendor disclosure; DAEMON Tools exploitation details under active investigation
- **Mitigation:** Remove and reinstall Nx Console from the official VS Code Marketplace; verify extension version against vendor release notes; uninstall affected DAEMON Tools Lite versions and monitor for credential misuse; apply vendor patches as released
- **Source:** [CISA KEV 2026-05-27](https://www.cisa.gov/news-events/alerts/2026/05/27/cisa-adds-three-known-exploited-vulnerabilities-catalog)

## Recommended Actions

- [ ] **SOC:** Hunt for CVE-2026-0300 exploitation by monitoring for unexpected shellcode injection signatures in PAN-OS nginx processes, outbound TCP/6081–6082 from firewall management interfaces, and unusual Active Directory enumeration sourced from firewall service accounts. Enable Palo Alto Threat Prevention signatures for CL-STA-1132 TTPs.
- [ ] **SOC:** Block all domains listed in the Phishing & Malware Distribution section at DNS and proxy layers. Add ThreatFox domains to SIEM watchlists. Monitor for TrapDoor C2 traffic to `ddjidd564.github.io`.
- [ ] **SOC:** Alert on any `.cursorrules` or `CLAUDE.md` file modifications in developer home directories not traceable to a known code commit — TrapDoor plants these to manipulate AI coding assistants.
- [ ] **IT / Patch Mgmt:** **Priority 1 —** Apply Palo Alto hotfix for CVE-2026-0300 on all PA-Series and VM-Series firewalls immediately; disable User-ID Authentication Portal if patching is delayed. **Priority 2 —** Apply FortiClient EMS hotfix (version 7.4.7) for CVE-2026-35616. **Priority 3 —** Enable Exchange Emergency Mitigation (EM) Service for CVE-2026-42897; apply permanent patch when released. **Priority 4 —** Upgrade Ghost CMS to ≥ 6.19.1 to remediate CVE-2026-26980 ClickFix injection vector.
- [ ] **Dev / AppSec:** Immediately audit all developer environments for TrapDoor package names (`prompt-engineering-toolkit`, `solidity-deploy-guard`, `defi-threat-scanner`, `dev-env-bootstrapper`, `wallet-security-checker`, `eth-security-auditor`, `move-analyzer-build`). Rotate GitHub tokens, npm tokens, and cloud API keys for any affected environments. Pin TanStack router to a clean version and regenerate all CI/CD secrets. Audit GitHub Actions `pull_request_target` workflow permissions across all repositories. Remove and reinstall the Nx Console VS Code extension.
- [ ] **Leadership:** Two unpatched zero-days (CVE-2026-0300, CVE-2026-42897) with confirmed active exploitation warrant emergency patching authorization. The TrapDoor supply-chain campaign poses elevated insider-threat risk to development pipelines; recommend notifying affected DevOps teams and initiating a credential hygiene sweep. Consider notifying clients with PAN-OS or Exchange on-prem deployments of the elevated risk posture.
