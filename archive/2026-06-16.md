> **Daily Brief.** North Korea-aligned crypto-targeting operations dominate today's collection window: JINX-0164's trojanized npm package (@velora-dex/sdk) and Kimsuky's AUDIOFIX/MiniRAT macOS malware chain both continue to compromise developer and executive endpoints in the digital-asset sector. CISA's KEV catalog additions this month — Mirasvit Magento cache warmer, Arista EOS, Chromium V8, and Cisco SD-WAN Manager — remain unpatched at many organizations, while the wormable Windows Kernel flaw (CVE-2026-45657) and the actively exploited Check Point VPN bypass (CVE-2026-50751, tied to Qilin ransomware) carry over as top patching priorities. The unauthenticated nginx-ui MCP takeover bug (CVE-2026-33032) still has thousands of exposed instances despite a March fix. Defenders should prioritize Windows/Check Point/Magento patching, npm dependency audits for crypto/Web3 projects, and macOS endpoint monitoring for AUDIOFIX/MINIRAT indicators.

## Executive Summary

- **JINX-0164** trojanized the npm package `@velora-dex/sdk@4.9.1` to drop the Go-based MINIRAT backdoor on macOS crypto developer machines, part of a broader LinkedIn-lure campaign against Web3 firms.
- **Kimsuky (North Korea)** is deploying the Python-based AUDIOFIX infostealer via spoofed Cisco Webex invites, followed by the MiniRAT Go backdoor for lateral movement — both share C2 domains with the JINX-0164 cluster.
- **CVE-2026-45657** (Microsoft Windows Kernel, CVSS 9.8, wormable use-after-free) and **CVE-2026-50751** (Check Point VPN auth bypass, CVSS 9.3, exploited by Qilin ransomware affiliates since May 7) remain the two highest-priority unpatched flaws in client environments.
- **CVE-2026-45247** (Mirasvit Full Page Cache Warmer for Magento, CVSS 9.8, unauth RCE via PHP `unserialize()`) was added to CISA KEV; any Magento 2 store running the extension should patch to 1.11.12 immediately.
- **CVE-2026-33032** (nginx-ui MCP endpoint auth bypass, CVSS 9.8) still has 2,600+ internet-exposed, unpatched instances three months after a fix shipped — full server takeover remains possible.
- **BlueNoroff (Lazarus subgroup)** continues GhostCall/GhostHire social-engineering campaigns against crypto executives and developers using fake Zoom/Teams "update" clients and trojanized GitHub take-home assessments.

## APT / Threat Actor Activity

### JINX-0164 (financially motivated, crypto-focused)

- **Targets:** Cryptocurrency exchanges, Web3 startups, and blockchain developers, primarily on macOS
- **TTPs:** LinkedIn-based social engineering with fake recruiter/investor personas (`T1566.002`); fake teleconference "update" client delivery (`T1204.002`); npm package trojanization for CI/CD-adjacent compromise (`T1195.001`); Go-based backdoor for persistence and remote command execution (`T1059`)
- **Recent activity:** On April 7, 2026, JINX-0164 modified version 4.9.1 of the public npm package `@velora-dex/sdk`, appending a base64-encoded command that fetched and executed a shell script delivering the MINIRAT backdoor. The group also runs parallel LinkedIn meeting-lure campaigns that drop the custom AUDIOFIX-family malware directly onto developer and executive macOS systems. Traffic is routed through Mullvad, Astrill, and ExpressVPN exit nodes to obscure attacker infrastructure.
- **Attribution confidence:** Medium — distinct cluster tracked by multiple vendors (Wiz, SafeDep, Corgea); overlaps in tooling/targeting with North Korea-nexus crypto-theft operations are noted but not yet firmly attributed to a known state actor
- **Sources:** [The Hacker News](https://thehackernews.com/2026/05/jinx-0164-targets-cryptocurrency-firms.html), [SafeDep](https://safedep.io/malicious-velora-dex-sdk-npm-compromised-rat/), [Corgea](https://corgea.com/research/velora-dex-sdk-npm-minirat-macos-backdoor)

### Kimsuky (aliases: Thallium, Black Banshee, Velvet Chollima, APT43)

- **Targets:** Corporate administrators, defense contractors, and South Korean organizations; secondary focus on cryptocurrency credential theft
- **TTPs:** Spoofed Cisco Webex meeting invitations as initial lure (`T1566.001`); Python-based macOS infostealer deployment (`T1555.001`, browser/keychain credential theft); Go-based backdoor for lateral movement (`T1021`); session hijacking of Discord/Slack/Telegram tokens (`T1528`)
- **Recent activity:** Kimsuky's AUDIOFIX infostealer harvests password manager entries, iCloud Keychain and browser credentials, SSH keys, console history, and cryptocurrency wallet data; once a foothold is established, the group drops the Go-based MiniRAT backdoor to move laterally across corporate networks. AUDIOFIX and MiniRAT share C2 infrastructure with the JINX-0164 cluster, including the domains `datahub.ink`, `cloud-sync.online`, and `byte-io.us`.
- **Attribution confidence:** High — consistent with historical Kimsuky/APT43 tradecraft per SC Media and multiple vendor reports; North Korean state nexus
- **Sources:** [SC Media](https://www.scworld.com/brief/north-korean-hackers-kimsuky-target-south-korea-with-new-malware-variants), [GBHackers](https://gbhackers.com/jinx-0164-uses-linkedin-lures/)

### @velora-dex/sdk — npm Package Trojanization (JINX-0164)

- **Attack type:** Malicious update / compromised maintainer access
- **Ecosystem:** npm
- **Package(s) affected:** `@velora-dex/sdk@4.9.1`
- **Impact:** A base64-encoded command appended to the package fetched and executed a remote shell script that deployed MINIRAT, a Go-based backdoor providing persistence and remote command execution on developer macOS workstations.
- **IOCs:** C2 domains `datahub.ink`, `cloud-sync.online`, `byte-io.us`; attacker traffic routed through Mullvad VPN, Astrill VPN, and ExpressVPN exit nodes
- **Mitigation:** Downgrade or remove `@velora-dex/sdk` version 4.9.1; verify installed version against the registry's current release; audit any macOS developer machine that installed the package for MINIRAT persistence (launch agents/daemons); rotate SSH keys, cloud tokens, and crypto wallet credentials on affected hosts; enable `npm audit signatures` and package-lock verification in CI.
- **Source:** [SafeDep](https://safedep.io/malicious-velora-dex-sdk-npm-compromised-rat/), [Corgea](https://corgea.com/research/velora-dex-sdk-npm-minirat-macos-backdoor)

## Recommended Actions

- [ ] **SOC:** Monitor for outbound connections to `datahub.ink`, `cloud-sync.online`, and `byte-io.us`; flag any macOS host exfiltrating to these domains as a probable AUDIOFIX/MINIRAT infection.
- [ ] **SOC:** Hunt for unauthorized MeshCentral installations and unexpected `acme-client` npm installs as indicators of ShinyHunters' PeopleSoft post-exploitation kit.
- [ ] **SOC:** Scan exposed nginx-ui instances for unauthenticated `/mcp_message` access attempts; treat any hit as a confirmed compromise indicator for CVE-2026-33032.
- [ ] **IT / Patch Mgmt:** Apply Microsoft June 2026 Patch Tuesday updates immediately, prioritizing CVE-2026-45657 (Kernel RCE) and CVE-2026-41091 (Defender EoP, actively exploited).
- [ ] **IT / Patch Mgmt:** Apply the Check Point hotfix for CVE-2026-50751 and disable IKEv1 on all VPN gateways; treat as urgent given confirmed Qilin ransomware exploitation.
- [ ] **IT / Patch Mgmt:** Patch Mirasvit Full Page Cache Warmer to v1.11.12 on any Magento 2 storefront; patch nginx-ui to 2.3.4+ if not already done.
- [ ] **Dev / AppSec:** Audit all npm projects for `@velora-dex/sdk@4.9.1`; remove and re-pin to a clean version; rotate SSH keys, cloud credentials, and crypto wallet secrets on any macOS workstation that installed the package.
- [ ] **Dev / AppSec:** Extend npm provenance/signature verification (`npm audit signatures`) to cover crypto/Web3-adjacent dependencies given repeated targeting of this ecosystem by JINX-0164 and BlueNoroff.
- [ ] **Leadership:** Crypto/Web3 business units should brief staff on GhostCall/GhostHire and JINX-0164 social-engineering patterns (fake investor calls, fake recruiter technical assessments) given sustained targeting of executives and developers in this sector.
