> **Daily Brief.** The threat landscape on 29 May 2026 is defined by three converging pressures: an escalating barrage of exploited network-edge and endpoint vulnerabilities spanning Cisco SD-WAN, Palo Alto PAN-OS, Microsoft Defender, and the Linux kernel; a historically broad software supply chain attack wave that poisoned hundreds of widely-used npm, PyPI, and Crates.io packages in the span of a week; and intensified nation-state campaigns from China-, North Korea-, and Russia-aligned APTs targeting strategic industries, defense contractors, and open-source developer tooling. Federal FCEB agencies face multiple active KEV deadlines. Developer and DevSecOps teams should treat the TrapDoor, Mini Shai-Hulud / TeamPCP, and Megalodon findings as urgent-priority supply chain incidents requiring immediate audit of CI/CD pipelines and lock-file integrity.

## Executive Summary

- **Cisco Catalyst SD-WAN (CVE-2026-20182, CVSS 10.0)** is under active exploitation by at least 10 distinct threat clusters, including UAT-8616 (ORB-linked), deploying web shells, cryptominers, and credential stealers; CISA has added it to KEV with a now-passed remediation deadline.
- **Palo Alto PAN-OS (CVE-2026-0300, CVSS 9.3)** is being exploited for root-level RCE against internet-exposed firewall management planes; over 5,800 vulnerable instances remain accessible online.
- **Microsoft Defender** contains two actively exploited zero-days (CVE-2026-41091 "RedSun" LPE, CVE-2026-45498 "UnDefend" DoS) added to CISA KEV with a June 3, 2026 federal deadline.
- **TeamPCP (Mini Shai-Hulud)** poisoned 404 malicious versions across 170+ npm and PyPI packages — including TanStack, Mistral AI SDK, and OpenSearch — in an unprecedented registry worm attack affecting an estimated 15 million users; a separate TrapDoor campaign followed, targeting crypto/DeFi/AI developers across npm, PyPI, and Crates.io.
- **Megalodon** injected malicious CI/CD workflows into 5,561 GitHub repositories in a six-hour automated blitz on May 18, exfiltrating secrets and cloud credentials to a single C2.
- **ESET APT Activity Report (Oct 2025–Mar 2026)** documents China-aligned FamousSparrow targeting Venezuelan oil infrastructure; North Korea's Lazarus/BlueNoroff backdooring the axios npm library; and Russia's Sandworm deploying new Rust-written wiper ZeroRays against Ukrainian and Polish critical infrastructure.

> Incidents affecting software build pipelines, open-source packages, CI/CD infrastructure, and code-signing infrastructure reported within the last 7 days.

### TeamPCP / Mini Shai-Hulud — npm & PyPI Registry Worm
- **Attack type:** CI/CD pipeline injection via Pwn-Request / GitHub Actions OIDC token theft; self-propagating registry worm
- **Ecosystem:** npm, PyPI
- **Package(s) affected:** 170+ npm packages / 2 PyPI packages; 404 malicious versions total. Key targets: entire `@tanstack/*` router ecosystem (42 packages), `mistralai==2.4.6`, `guardrails-ai==0.10.1`, UiPath automation (65 packages), OpenSearch npm (1.3M weekly downloads)
- **Impact:** Credential harvesting (GitHub tokens, npm tokens, CI/CD secrets, cloud credentials, API keys). The worm propagated by committing poisoned `.claude/settings.json` and `.vscode/tasks.json` files to victim repositories via GitHub GraphQL API, spreading to downstream consumers.
- **IOCs:** Typosquat domain `git-tanstack.com`; C2 IP `83.142.209.194`; Dune-themed GitHub repository dead drops used for token exfiltration; getsession.org seed nodes used for C2
- **Mitigation:** Audit all `@tanstack/*`, `mistralai`, `guardrails-ai`, and UiPath npm/PyPI packages in use. Regenerate any CI/CD secrets or cloud credentials in affected environments. Inspect `.claude/` and `.vscode/` config files in repositories for unauthorized modifications. Review GitHub Actions workflow permissions.
- **Source:** [THN](https://thehackernews.com/2026/05/mini-shai-hulud-worm-compromises.html), [SafeDep](https://safedep.io/mass-npm-supply-chain-attack-tanstack-mistral/), [NHS England](https://digital.nhs.uk/cyber-alerts/2026/cc-4781)

### TrapDoor — Cross-Ecosystem Credential Stealer (npm / PyPI / Crates.io)
- **Attack type:** Typosquatting; postinstall hooks; import-time execution; build.rs scripts (Rust); remote payload delivery via attacker-controlled GitHub Pages
- **Ecosystem:** npm, PyPI, Crates.io
- **Package(s) affected:** 34 malicious packages across 384 versions. npm packages include: `async-pipeline-builder`, `crypto-credential-scanner`, `defi-threat-scanner`, `eth-wallet-sentinel`, `llm-context-compressor`, `mnemonic-safety-check`, `solidity-deploy-guard`, `wallet-security-checker`, `web3-secrets-detector`. Rust crates: `move-analyzer-build`, `sui-framework-helpers`, `sui-sdk-build-utils`. PyPI: `cryptowallet-safety`, `eth-security-auditor`, `git-config-sync`, `solidity-build-guard`.
- **Impact:** Steals developer secrets, crypto wallets, SSH keys, cloud credentials, browser data, environment variables. npm payload (`trap-core.js`) validates stolen AWS and GitHub tokens via API calls. Rust crates encrypt and exfiltrate keystores to GitHub Gists. Persistence via cron, systemd, Git hooks, shell hooks, `.cursorrules`, and `CLAUDE.md` files. Novel technique: PR injection into popular AI repos (`browser-use/browser-use`, `langchain-ai/langchain`) with hidden instructions targeting AI coding assistants.
- **IOCs:** C2/payload delivery domain `ddjidd564.github.io`; earliest activity 2026-05-22 20:20 UTC
- **Mitigation:** `npm uninstall <package>` / `pip uninstall <package>` / `cargo remove <crate>` for any listed packages. Regenerate all cloud credentials, SSH keys, and GitHub tokens in affected environments. Review `CLAUDE.md` and `.cursorrules` files in repositories for hidden instructions. Inspect Git hooks in CI/CD pipelines.
- **Source:** [THN](https://thehackernews.com/2026/05/trapdoor-supply-chain-attack-spreads.html), [Socket.dev](https://socket.dev/blog/trapdoor-crypto-stealer-npm-pypi-crates)

### Megalodon — GitHub Actions Mass CI/CD Workflow Poisoning
- **Attack type:** Automated CI/CD workflow injection at scale; GitHub Actions secret exfiltration
- **Ecosystem:** GitHub Actions
- **Package(s) affected:** 5,561 GitHub repositories (5,718 commits injected in 6-hour window on 2026-05-18, 11:36–17:48 UTC)
- **Impact:** Malicious workflows exfiltrate CI secrets, cloud credentials, SSH keys, OIDC tokens, and source code secrets to attacker C2. Two variants: SysDiag (mass, triggered on every push/PR) and Optimize-Build (targeted, dormant workflow_dispatch backdoor).
- **IOCs:** C2 server `216.126.225.129:8443`; forged commit author emails `build-system@noreply.dev`, `ci-bot@automated.dev`; author names `build-bot`, `auto-ci`, `ci-bot`, `pipeline-bot`
- **Mitigation:** Search GitHub organization repositories for commits from `build-system@noreply.dev` or `ci-bot@automated.dev`. Audit workflow files added between 11:36–17:48 UTC on 2026-05-18. Rotate all secrets stored in GitHub Actions. Review `GITHUB_TOKEN` permissions and branch protection rules. Consider StepSecurity Harden-Runner for workflow monitoring.
- **Source:** [THN](https://thehackernews.com/2026/05/megalodon-github-attack-targets-5561.html), [StepSecurity](https://www.stepsecurity.io/blog/megalodon-mass-github-actions-secret-exfiltration-across-5-500-public-repositories), [SafeDep](https://safedep.io/megalodon-mass-github-repo-backdooring-ci-workflows/)
