- **CVE-2026-0257 (Palo Alto PAN-OS GlobalProtect, CVSS 9.1)** is confirmed actively exploited with CISA KEV deadline passed June 1; two distinct attack waves from Vultr and Dromatics Systems infrastructure have compromised networks via forged authentication override cookies.

## APT / Threat Actor Activity

### Screening Serpens (aliases: UNC1549, Smoke Sandstorm, Iranian Dream Job)

- **Targets:** Aerospace, defense manufacturing, telecommunications, and technology sector organizations in the US, Israel, UAE, and at least two additional Middle Eastern countries
- **TTPs:** Spear-phishing via fake job requisitions impersonating air carriers and recruitment platforms; DLL sideloading via legitimate .NET binaries; **AppDomainManager hijacking** to disable ETW telemetry and bypass EDR before payload execution (`T1574.014`, `T1574.001`); task scheduler persistence (`T1053.005`); chunked data exfiltration to Azure-hosted C2 infrastructure (`T1041`)
- **Recent activity:** Between February and April 2026, deployed six RAT variants across two families — **MiniUpdate** (US, Israel, UAE, Middle East campaigns) and **MiniJunk V2** (Middle East, US campaigns). Activity aligned with onset of regional Middle East conflict (February 28, 2026). C2 infrastructure uses Azure Web Apps mimicking legitimate Windows service and enterprise brand names. First confirmed use of AppDomainManager configuration files to natively disable .NET CLR security mechanisms, including ETW suppression (`<etwEnable enabled="false"/>`) and strong-name validation bypass.
- **Attribution confidence:** Moderate-High — Unit 42 assessment based on shared file naming conventions, DLL sideloading chains, ONLYOFFICE DocSpace delivery infrastructure, and C2 domain patterns consistent with prior Screening Serpens operations documented by Check Point Research and ClearSky
- **Sources:** [Unit 42 — Tracking Screening Serpens' 2026 Espionage Campaigns](https://unit42.paloaltonetworks.com/tracking-iran-apt-screening-serpens/)

### @redhat-cloud-services npm Packages — "Miasma: The Spreading Blight"

- **Attack type:** Compromised maintainer account / malicious preinstall hook / self-propagating credential-stealing worm
- **Ecosystem:** npm
- **Package(s) affected:** 32 packages across the @redhat-cloud-services scope, including `@redhat-cloud-services/vulnerabilities-client`, `@redhat-cloud-services/tsc-transform-imports`, `@redhat-cloud-services/topological-inventory-client`, `@redhat-cloud-services/sources-client`, `@redhat-cloud-services/rbac-client`, and others — 96 compromised versions total; ~116,991 weekly downloads; ~9.8 million total downloads cumulative
- **Impact:** Obfuscated 4.2 MB JavaScript payload executes via `preinstall` lifecycle hook before any application code. Sweeps GitHub Actions secrets, npm/PyPI publish tokens, AWS access keys and session tokens, GCP application default credentials and service account keys, Azure service principal credentials, HashiCorp Vault tokens, Kubernetes service accounts and kubeconfig files, SSH private keys, Docker registry credentials, GPG keys, and all `.env` files. Self-propagating: stolen publish tokens used to inject worm into additional packages. Establishes persistence via `~/.claude/settings.json` (Claude Code) and `tasks.json` with `"runOn": "folderOpen"` (VS Code) so malware re-executes on every session. Avoids execution on Russian-language systems.
- **IOCs:** Exfiltration endpoint: `api.anthropic[.]com:443/v1/api` (spoofed/tunneled); GitHub used as fallback exfiltration channel with commit messages including `IfYouInvalidateThisTokenItWillNukeTheComputerOfTheOwner:<token>`; GitHub repo descriptions: "Miasma: The Spreading Blight"
- **Mitigation:** (1) Identify and isolate any host that installed affected `@redhat-cloud-services` versions; (2) Rotate all secrets accessible from affected environments — GitHub tokens, AWS keys, GCP/Azure credentials, Kubernetes service accounts, SSH keys, npm tokens; (3) Audit `~/.claude/settings.json`, `.vscode/tasks.json`, `.github/workflows/`, `.github/setup.js` for persistence artifacts; (4) Suspend affected CI/CD workflow runs and invalidate build artifacts from the exposure window; (5) Do not treat `npm uninstall` or deleting `node_modules` as sufficient cleanup
- **Source:** [The Hacker News](https://thehackernews.com/2026/06/miasma-supply-chain-attack-compromises.html), [Socket](https://socket.dev/blog/mini-shai-hulud-campaign-hits-red-hat-cloud-services-npm-packages), [Orca Security](https://orca.security/resources/blog/red-hat-npm-supply-chain-attack/)

### Mini Shai-Hulud Worm — Ongoing npm/PyPI Ecosystem Activity (TeamPCP)

- **Attack type:** Self-propagating supply chain worm / compromised maintainer accounts / CI/CD pipeline injection
- **Ecosystem:** npm, PyPI
- **Package(s) affected:** 320+ npm packages confirmed in the latest wave (SecurityWeek reporting); previous waves hit xinference (PyPI), litellm (PyPI), pgserve (npm), Checkmarx KICS Docker images and VS Code extensions, Nx Console (CVE-2026-48027), TanStack (CVE-2026-45321)
- **Impact:** Credential theft from developer machines and CI/CD pipelines; SLSA Build Level 3 provenance attestation forgery; OIDC token extraction from GitHub Actions runner process memory; downstream package poisoning via stolen publish tokens
- **IOCs:** ICP canister-backed C2 channel (CanisterSprawl variant); plain tar.gz exfiltration to attacker C2 (xinference variant)
- **Mitigation:** Audit npm and PyPI dependencies for unexpected version bumps; verify lock files; enforce npm publish token scoping; review GitHub Actions OIDC token permissions; use StepSecurity Harden-Runner or equivalent CI hardening
- **Source:** [GitGuardian](https://blog.gitguardian.com/three-supply-chain-campaigns-hit-npm-pypi-and-docker-hub-in-48-hours/), [SecurityWeek](https://www.securityweek.com/over-320-npm-packages-hit-by-fresh-mini-shai-hulud-supply-chain-attack/)

## Recommended Actions

- [ ] **SOC:** Hunt for PAN-OS GlobalProtect exploitation: search VPN logs for sessions authenticated with override cookies targeting local administrator accounts, especially connections originating from Vultr IP ranges or Dromatics Systems infrastructure. Correlate with CVE-2026-0257 Rapid7 IOC list.
- [ ] **SOC:** Hunt for Screening Serpens indicators: search endpoint telemetry for AppDomainManager config files containing `<etwEnable enabled="false"/>`, scheduled tasks named `WindowsSecurityUpdate` or `Synchronize OS`, and DLL loads of `UpdateChecker.dll`, `uevmonitor.dll`, or `Connection.dll`.
- [ ] **SOC:** Hunt for Miasma worm persistence: check all developer machines and CI/CD runners for artifacts in `~/.claude/settings.json`, `.vscode/tasks.json` with `"runOn": "folderOpen"`, `.github/workflows/codeql.yml`, and `.github/setup.js`. Treat any unexpected entries as Indicators of Compromise.
- [ ] **IT / Patch Mgmt:** **URGENT** — Verify Microsoft Defender Malware Protection Engine version 1.1.26040.8 or later across all Windows endpoints before June 3 CISA KEV deadline (CVE-2026-41091, CVE-2026-45498, CVE-2026-45584).
- [ ] **IT / Patch Mgmt:** Apply Trend Micro Apex One SP1 CP Build 18012 (on-premise) or agent build 14.0.20731 (SaaS) before June 4 CISA KEV deadline (CVE-2026-34926).
- [ ] **IT / Patch Mgmt:** Apply PAN-OS patch for CVE-2026-0257 immediately if not yet done. Post-patch: audit GlobalProtect gateway logs for VPN sessions with override cookie authentication, particularly those using local administrator accounts.
- [ ] **IT / Patch Mgmt:** Update Spectra Gutenberg Blocks to ≥2.19.26, GEO my WP to ≥4.5.6, and Simple History to ≥5.26.1 on all WordPress deployments.
- [ ] **IT / Patch Mgmt:** Update Exim to 4.99.3 (GnuTLS builds only). Do not expose ChromaDB API servers to the internet pending official patch.
- [ ] **Dev / AppSec:** Audit all npm dependencies for unexpected `@redhat-cloud-services` package versions. Rotate all secrets accessible from any environment that installed the affected packages. Review lock files and CI/CD pipeline configurations for injected workflow steps.
- [ ] **Dev / AppSec:** Enable npm package provenance verification and restrict publish token scopes. Enforce `npm audit` and Sigstore attestation checks in CI pipelines to detect Mini Shai-Hulud variants.
- [ ] **Leadership:** The Miasma supply chain worm represents a systemic risk to any organization using Red Hat Hybrid Cloud Console tooling or consuming affected packages as transitive dependencies — assess your exposure before the next sprint. Consider mandatory secrets rotation as a precautionary measure across developer teams.
- [ ] **Leadership:** With the FIFA World Cup 2026 beginning, advise employees and any consumer-facing teams about GHOST STADIUM phishing. Enforce MFA on all FIFA/sports platform accounts; alert finance teams to watch for transactions through fraud payment gateways.
