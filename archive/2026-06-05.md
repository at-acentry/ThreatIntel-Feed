- **CVE-2026-8206** (Kirki WordPress plugin, CVSS 9.8) enables unauthenticated account takeover on 500,000+ sites; ~150,000 remain unpatched.

**CVE-2026-45247 Detail:** Affects Mirasvit Full Page Cache Warmer <1.11.12. The extension deserializes user-controlled data from the `CacheWarmer` cookie via PHP's native `unserialize()` without class-instantiation restrictions, enabling unauthenticated RCE via gadget chain. Observed payloads use base64-encoded serialized PHP objects. Patch to v1.11.12 released 2026-05-25. KEV due date: 2026-06-06.

**CVE-2026-8206 Detail:** Kirki REST endpoint `handle_forgot_password()` accepts an arbitrary attacker-controlled email for any valid username, allowing full account takeover without authentication. Over 500,000 sites affected; ~150,000 remain unpatched as of publication. Patch available in Kirki 6.0.7.

**CVE-2024-21182 Detail:** Oracle WebLogic T3/IIOP protocol attack surface; unauthenticated network-accessible exploit. Patched in Oracle's July 2024 CPU; KEV addition confirms significant unpatched population remains in production. Federal agencies required to remediate by 2026-06-04.

## Software Supply Chain Attacks

### @redhat-cloud-services npm Packages — Miasma Worm (June 1, 2026)
- **Attack type:** Compromised maintainer account (GitHub employee credential compromise); malicious `preinstall` lifecycle hook injection
- **Ecosystem:** npm
- **Package(s) affected:** 32 packages under `@redhat-cloud-services` namespace; 96 malicious versions published in two waves (10:53 UTC and 13:44–13:46 UTC on 2026-06-01); ~116,991 cumulative weekly downloads. Confirmed affected: `@redhat-cloud-services/chrome` v2.3.1/2.3.2/2.3.4, `@redhat-cloud-services/frontend-components` v7.7.2/7.7.3/7.7.5, `@redhat-cloud-services/types` v3.6.1/3.6.2/3.6.4
- **Impact:** 4.2 MB obfuscated JavaScript payload executes at install time before application code; sweeps 86 environment variables and 20 credential files targeting AWS keys/session tokens, GCP ADC and service account keys, Azure service principal creds, HashiCorp Vault tokens, Kubernetes kubeconfig and service accounts, npm/PyPI publish tokens, SSH private keys, Docker registry credentials, GPG keys, and all `.env` files; exfiltrates via GitHub account `liuende501` (236 dead-drop repositories); Miasma variant of Shai-Hulud with improved static analysis evasion
- **IOCs:** Hardcoded AES-128-GCM keys: `fe0d71d57ecf4fa0a433185bf59a03f5`, `f5e5dca9b725ec18514c4b322ed35d2b`; `binding.gyp` SHA-256: `ef641e956f91d501b748085996303c96a64d67f63bfeef0dda175e5aa19cca90`; exfil dead-drop GitHub org: `liuende501`
- **Mitigation:** Remove all compromised versions immediately; run `npm audit` and compare lock files against published safe versions; rotate all secrets exposed in the scope above (especially cloud IAM, SSH, and CI/CD tokens); audit GitHub Actions workflows for injected steps; pin safe known-good versions in `package-lock.json`
- **Source:** [Wiz Blog](https://www.wiz.io/blog/miasma-supply-chain-attack-targeting-redhat-npm-packages), [Microsoft Security Blog](https://www.microsoft.com/en-us/security/blog/2026/06/02/preinstall-persistence-inside-red-hat-npm-miasma-credential-stealing-campaign/), [Aikido Security](https://www.aikido.dev/blog/red-hat-npm-packages-compromised-credential-stealing-worm)

### IronWorm — npm Web3/Crypto Developer Targeting (June 3, 2026)
- **Attack type:** Compromised maintainer account (`asteroiddao` GitHub account); Rust-written worm with self-propagation via stolen npm publish tokens
- **Ecosystem:** npm
- **Package(s) affected:** 36 packages primarily targeting Web3/crypto developer organizations including ocrybit, asteroid-dao, alisista, warashibe, kakedashi-hacker, weavedb, ArweaveOasis
- **Impact:** 976 KB Rust ELF binary executed via `preinstall` hook; packed with custom UPX stub to defeat signature detection; per-call-site encrypted strings; carries embedded eBPF kernel-level rootkit for persistence; targets 86 environment variables and 20 credential files including OpenAI, AWS, Anthropic, npm tokens, SSH keys, vault configs, and Exodus wallet files; self-propagates by injecting into victims' GitHub repos and publishing trojanized packages to npm registry
- **IOCs:** Initial compromise vector: `asteroiddao` npm/GitHub account; eBPF rootkit drops kernel module — monitor for unexpected `bpf()` syscalls post-install
- **Mitigation:** Immediately remove affected packages and regenerate all exposed credentials; audit `preinstall` and `postinstall` scripts in all dependencies; use `npm ci` with integrity checks; consider `--ignore-scripts` flag for install-time risk reduction; rotate npm publish tokens
- **Source:** [JFrog Security Research](https://research.jfrog.com/post/iron-worm-shai-hulud-rustier-cousin/), [BleepingComputer](https://www.bleepingcomputer.com/news/security/new-ironworm-malware-hits-36-packages-in-npm-supply-chain-attack/), [Dark Reading](https://www.darkreading.com/cyberattacks-data-breaches/rust-written-ironworm-npm-supply-chain)
